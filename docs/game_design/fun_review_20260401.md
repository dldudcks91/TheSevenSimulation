## 1. 현재 기획의 문제점

### 문제 1: 사기 양극단이 모두 나쁨 → "안정 유지 게임"

**현재**: 사기 0 = 이탈(나쁨), 사기 100 = 폭주(나쁨). 양쪽 극단이 모두 위험.
**결과**: 플레이어의 최적 전략 = 모든 영웅을 40~60 사이에 유지 = **게임에서 가장 지루한 상태**.

드라마를 만들겠다는 시스템이 드라마를 회피하게 만드는 구조.
"연쇄 반응이 핵심 재미"라고 하면서, 연쇄 반응의 모든 결과가 부정적이다.

→ **해결: 사기는 높을수록 좋되, 100이 되면 폭주. 2장에서 상세.**

### 문제 2: 7명 × 죄종 1개 = 창발이 아니라 예측 가능한 이벤트 트리

- 영웅 7명, 죄종 7종(각 1개) = 조합 공간이 극도로 제한적
- 30개 이벤트 × 3~4선택지 = 총 약 100개 선택지. 1~2회차면 전부 봄
- 폭주 이벤트 죄종당 1개, 이탈 이벤트 죄종당 1개 = 매번 같은 사건
- 연쇄 반응 규칙 7개 = 매번 같은 패턴

**RimWorld가 1000시간 가능한 이유**: 주민 20+명 × 수십 개 특성 × 수백 개 이벤트 × 무작위 조합.
**이 게임**: 7명 × 1특성 × 30이벤트 = 2~3회차면 패턴이 보임.

→ **해결: 독립 세부스탯 7개를 행동에 적극 반영. 3장에서 상세.**

### 문제 3: 전투가 3종류인데 3종류 다 재미없음

| 전투 유형 | 플레이어 행동 | 실제 체감 |
|----------|------------|----------|
| 원정 | 파티 편성 → 카드 사용 → 관전 | 같은 오토배틀 반복 |
| 방어 | 카드 사용 → 관전 | 같은 오토배틀 반복 |
| 사냥 | 관전만 (카드 불가) | 구경만 |

카드가 소모품 → 아끼려는 심리 → 작은 전투에서 사용 안 함 → 관전만.
일기토 "자동 발동" → 가장 극적인 순간에 플레이어가 개입 불가.

→ **해결: 카드 제거, 지시+죄종 기술 시스템으로 대체. 4장에서 상세.**

### 문제 4: "판결자" 판타지가 실제로는 "시프트 매니저" 판타지

실제 게임에서 하는 일:
- 아침: 이벤트 선택지 클릭 (수동적 반응)
- 낮: 7명을 10개 슬롯에 배정 (스케줄 관리)
- 저녁: 계속/철수 바이너리 선택
- 밤: 오토배틀 관전 + 카드 클릭

마왕이라는 느낌이 전혀 안 남. 바알이 능동적으로 할 수 있는 일이 없음.

→ **해결: 포고령을 비가역적 분기 트리로, 전투에서 바알 지시 시스템 추가. 4장, 5장에서 상세.**

### 문제 5: 경제가 "딜레마"가 아니라 "정답 찾기"

- 식량: 채집 안 하면 망함 → 채집 1~2명 고정 (강제 루틴)
- 나무: 벌목 안 하면 건설 못 함 → 벌목 1명 고정 (강제 루틴)
- 매 턴 같은 배분 반복 → 자원 관리가 아니라 "의무 이행"

→ **해결: 6장에서 상세.**

### 문제 6: 죄종 만족/불만이 "체크리스트"

| 죄종 | 만족 조건 | 플레이어 행동 |
|------|----------|------------|
| 분노 | 원정 파견 | 3일마다 원정에 넣기 |
| 나태 | 시설 미배치 | 쉬게 두기 |
| 폭식 | 장비 풀장착 | 장비 채우기 |
| 색욕 | 선호 동료 배치 | 같이 배치하기 |

대부분 정답이 명확. 시기/교만은 플레이어 통제 불가능.

→ **수정 예정 (별도 논의)**

### 문제 7: 중후반 콘텐츠 고갈

Day 15쯤이면:
- 건물 대부분 완성, 이벤트 20개+ 경험, 카드 5종 보유, 영웅 7명 확보
- 새로운 시스템이 열리지 않음
- 후반 10~15턴 = "같은 일 반복 + 숫자만 커지는 적"

### 문제 8: 연쇄 반응 결과가 전부 "사기 하락"

연쇄 반응 7개 중 5개가 "사기 -N". 결과가 다양하지 않음.
RimWorld는 각 단계가 다른 종류의 문제를 만듦. 이 게임은 "사기 숫자가 내려간다"로 수렴.

→ **해결: 8장에서 상세.**

### 문제 9: 원정 "더 진행?" 판단에 정보 부족

다음 스테이지 난이도를 전혀 모름 → "더 진행?"은 도박이지 전략이 아님.
FTL은 다음 섹터의 대략적 위험을 보여주고, DD는 던전 맵을 점진 탐색.

### 문제 10: 방어 배치 딜레마가 습격 예고 날에만 작동

감시탑이 예고해주면: 습격 없는 날 → 방어 0명(정답), 습격 있는 날 → 일부 방어(강제).
"매 턴 고민"이 아니라 "이진 분기".

## 2. 사기 시스템 재설계 ✅ 확정 (2026-04-02) → CSV+코드 반영 완료 (2026-04-13, 구간: 0/1~25/26~75/76~99/100)

### 방향: 높을수록 좋음, 100 = 폭주

RimWorld의 Mood 모델 참고:
- Mood 높음 → Inspiration(영감) 확률 발동 (긍정적 폭발)
- Mood 낮음 → Mental Break 확률 발동 (부정적 폭발)
- 플레이어는 Mood를 높이고 싶다. 높이는 과정이 재미.

### 확정: 5구간 사기 체계

| 구간 | 상태 | 효과 |
|------|------|------|
| 0 | **이탈** | 거점을 떠남 |
| 1~30 | 불만 | 능력 저하 + **이탈 확률 체크** |
| 31~70 | 안정 | 정상 운용 |
| 71~99 | **고양** | 능력 강화 + **구원(Inspiration) 확률 발동** + **폭주 확률 체크** |
| 100 | **폭주** | 통제 불능 |

> 폭주 확률 %, 구원 발동 조건, 능력 강화/저하 수치는 밸런싱 단계에서 확정.

### 확정: 사기 변동 로그 (Modifier 대신)

RimWorld식 modifier 합산 방식은 **7명 25턴 규모에서 오버엔지니어링**으로 판단.
- RimWorld(20명+, 수백 시간)에서 modifier가 필요한 이유 = 추적 불가능한 규모
- DD(4명, 던전 단위)/CK3(1명, 이벤트 구동) = modifier 없이도 극적인 시스템 구현
- 이 게임(7명, ~25턴) = 플레이어가 직접 판결/배치/파견 → 원인을 이미 앎

**채택안: 단일 수치 + 변동 로그**
- 사기는 단일 숫자(0~100) 유지
- 사기 버튼 클릭 → 팝업에 **최근 7건 변동 로그** 표시
- 죄종/세부스탯에 따라 **같은 사건 → 다른 반응** (CK3식 성격 반응)

```
┌─ 아론 (분노) 사기: 63 ──────┐
│  원정 승리          +10      │
│  동료 부상 목격     -3       │
│  유리한 판결        +8       │
│  식량 부족          -5       │
│  시설 배치          +3       │
│  ...                         │
└──────────────────────────────┘
```

## 3. RimWorld 특성 시스템 분석 + 적용 — 논의 중 (2026-04-02)

### RimWorld 특성 시스템 상세 조사 (2026-04-02)

#### 기본 구조
- 주민당 **2~3개 특성** (단어 형태, 숫자가 아님)
- 총 **~60개 바닐라 특성** (스펙트럼 30 + 독립 30)
- 생성 시 가중 랜덤 배정, 이후 변경 불가
- 상충 특성은 동시 배정 차단 (Kind + Psychopath 불가)

#### 스펙트럼 특성 (9개 축, 각 2~4단계)
| 스펙트럼 | 단계 (긍정 → 부정) |
|---------|-------------------|
| 근면성 | Industrious(+35%) → Hard Worker(+20%) → Lazy(-20%) → Slothful(-35%) |
| 속도 | Jogger → Fast Walker → Slowpoke |
| 기본 기분 | Sanguine(+12) → Optimist(+6) → Pessimist(-6) → Depressive(-12) |
| 멘탈 | Iron-Willed(-18%) → Steadfast(-9%) → Nervous(+8%) → Volatile(+15%) |
| 외모 | Beautiful(+40 호감) → Pretty(+20) → Ugly(-20) → Staggeringly Ugly(-40) |

#### 핵심 원리 — 하나의 특성이 여러 시스템에 동시 영향

| 특성 | Mood | 작업 | 관계 | Mental Break | 전투 |
|------|------|------|------|-------------|------|
| Bloodlust(혈전광) | 킬 시 +8, +13 | - | 소셜파이트 4배 | Berserk 빈도↑ | 근접 보너스 |
| Kind(친절) | - | 간수 보너스 | 전원 +5 mood | 모욕 불가 | - |
| Pyromaniac(방화광) | 불 근처 mood↑ | **소방 불가** | - | **방화 전용** Break | - |
| Psychopath | 사망/장기적출 무시 | - | 긍정 사교 무시 | - | - |
| Tough | - | - | - | - | **피해 -50%** |
| Very Neurotic | - | **+40% 작업속도** | - | +14% 임계 | - |

#### 핵심 원리 — 같은 사건, 다른 반응 (특성이 Thought를 무효화/반전)

```
동료가 죽었다:
  일반 → Mood -25
  Psychopath → 아무 영향 없음 (무효화)
  Bloodlust → 영향 없음 (무효화)

인육 식사:
  일반 → Mood -20
  Cannibal → Mood +20 (반전!)

좋은 침실 배정:
  일반 → Mood +5
  Ascetic → Mood -5 (반전! 화려한 방 싫어함)
```

#### 핵심 원리 — 특성 조합이 만드는 창발적 드라마

```
Pyromaniac + Industrious + Kind
  → 평소: 최고 일꾼 + 모두에게 친절
  → Mood 하락 시: 방화 → 자기가 만든 시설 전소 → 소방 불가 → 전체 붕괴

Very Neurotic + Industrious = +75% 작업속도
  → 생산 괴물, 하지만 멘탈 임계 +14% → 자주 폭발
  → Sanguine(+12 mood) 추가되면 상쇄 → 완벽한 일꾼

Abrasive + Bloodlust = 소셜 폭탄
  → 모욕 2배 × 싸움 4배 → 주변 전원 mood↓ → 연쇄 멘탈브레이크
```

#### RimWorld가 작동하는 이유 요약
1. 특성은 **2~3개 단어** → 읽으면 바로 이해됨 (숫자 7개가 아님)
2. 한 특성이 **여러 시스템을 교차** → 예측 불가능한 상호작용
3. **같은 사건에 다른 반응** → 주민마다 다른 드라마
4. 특성은 **양면성** → 완벽한 조합 없음, 항상 트레이드오프

### 우리 게임 현황과 차이

| | RimWorld | TheSevenSimulation |
|---|---------|-------------------|
| 인원 | 20명+ | 7명 |
| 특성 | 2~3개 **단어** | 7개 **숫자** (세부스탯 1~20) |
| 조합 읽기 | "Bloodlust, Kind" → 즉시 이해 | "공격성16, 감수성9, 독립심12..." → 파악 불가 |
| 플레이 시간 | 수백 시간 | ~25턴/챕터 |

### 초안 제안 — 세부스탯 7개 전부 활용

| 세부스탯 | 높으면 (15+) | 낮으면 (5-) |
|---------|-------------|------------|
| 공격성 | 전투 ATK↑, **갈등 이벤트 빈도 2배**, 폭주 시 "파괴" 유형 | 전투 소극, 거점 내 평화 |
| 욕심 | 보상 획득량↑, **자원 갈등 이벤트**, 도난 확률↑ | 수확 적지만 나눔 |
| 자존심 | 리더 시 능력↑, **부하 역할 거부 확률**, 칭찬 시 사기++, 질책 시 사기--- | 어디든 배치 가능 |
| 호기심 | 탐색/연구 보너스, **위험 이벤트 조우↑** | 안전하지만 발견 기회 적음 |
| 집요함 | 장기 작업 보너스, **중단/철수 판단 느림** | 판단 빠르지만 오래 못 버팀 |
| 감수성 | 관계 형성 빠름, **다른 영웅 사기에 연동됨**, 연쇄 전염 취약 | 냉정, 연쇄 전염 저항 |
| 독립심 | 혼자 작업 효율↑, **팀 배치 시 효율↓** | 팀에서 잘 맞지만 혼자선 약함 |

### 대안 제안 — 수식어(상위2) 중심 (2026-04-02 논의)

RimWorld 분석 결과, 특성의 핵심은 **읽기 쉬운 단어**가 **여러 시스템을 교차**하는 것.
숫자 7개를 전부 활용하면 RimWorld와 정반대 방향(스프레드시트)으로 감.

**제안**: 기존 수식어 시스템(독립 세부스탯 상위 2개 조합 = 21종)을 **행동 분기 + 이벤트 트리거**로 승격.

```
죄종 7 × 수식어 21 = 147가지 영웅 유형

같은 분노 영웅이라도:
  "약탈자(공격성+욕심)" → 전투 후 전리품 독점 이벤트
  "고독한(감수성+독립심)" → 혼자 사냥 효율↑, 팀 전투 마찰
  "폭군(공격성+자존심)" → 리더 아니면 불만, 리더면 극강
```

> **미결**: 7개 전부 활용 vs 상위 2개(수식어) 중심 — 추후 확정

## 4. 카드 시스템 제거 → 대체안 ✅ 카드 제거 + 죄종 반응 방향 확정 (2026-04-13, 코드 반영 완료)

### 카드의 문제

- 소모품 → 아끼려는 심리 → 작은 전투에서 미사용 → 관전만
- 5종은 단순 버프/디버프 → 전략적 깊이 없음
- 보스전에 몰아 쓰기 → 압승 or 실패

### 대안 A: 바알의 지시(Directive) 시스템 — 삼국지9/14 방식

```
전투 전: 영웅별 "전투 방침" 설정
  [돌격] ATK +30%, DEF -20%
  [수비] DEF +30%, ATK -20%
  [유인] 후퇴하며 적을 끌어옴
  [일기토] 일기토 적극 신청

전투 중: "지시" 내림 (재사용 가능, 쿨다운)
  [진격] → 전원 공격적 전환 (쿨 5틱)
  [후퇴] → 전원 수비적 전환
  [집중 공격] → 영웅 지목 → 특정 적 집중
  [증원] → 대기 영웅 투입 (방어전)
```

장점: 소모되지 않음 → 매 전투 능동 참여. 쿨다운 = 타이밍 전략.

### 대안 B: 진형/포메이션 시스템 — HoMM3 방식

```
전투 전: 3명 영웅의 "진형" 선택
  [쐐기형] → 선봉 1명 ATK↑, 후방 2명 보호
  [횡대형] → 균등 배분, 범위 공격에 약함
  [학익진] → 좌우 협공, 중앙 약점

전투 중: "진형 전환"으로 대응
```

### 대안 C: 죄종 발동 시스템

카드 대신 **영웅의 죄종이 전투 중 능력으로 발현**:

| 죄종 | 기술 | 효과 | 리스크 |
|------|------|------|--------|
| 분노 | 광전사 | ATK 2배, 3틱 | 아군 구분 없이 공격 |
| 교만 | 불굴 | 3틱 무적 | 이후 1틱 경직 |

| 나태 | 안식 | 아군 전원 HP 지속 회복 | 이동 속도 -50% |
| 색욕 | 유혹 | 가장 약한 적 1체 일시 아군화 (3틱) | 약한 적만 대상 |

**발동 조건 — 사기와 연동:**
- 사기 71+(고양): 죄종 기술 **자동 발동** 확률 UP (강력하지만 통제 불능)
- 사기 31~70(안정): 플레이어가 SP로 **수동 발동** (통제 가능)
- 사기 1~30(불만): 죄종 기술 **발동 불가** (의욕 없음)

## 5. Frostpunk 분석 + 적용

### 법률(Book of Laws) 시스템

Frostpunk의 법률은 **비가역적 분기 트리**. 각 법률은 영구적이고 되돌릴 수 없음.

**대표적 법률 체인:**

```
아동 관련:
  아동 보호소 (안전, +Hope)
    → 아동 노동 (인력↑, +Discontent)
    → 또는 견습 엔지니어 (지연 보상)

신앙 트리:
  기도의 집 → 저녁 기도 → 성전 → 신앙 수호대(비밀경찰) → 새로운 신앙(신정정치)

질서 트리:
  경비대 → 순찰 → 감시탑 → 감옥 → 선전 → 새로운 질서(독재)
```

**왜 작동하나:**
- 법률은 **현재 위기를 해결**하지만 **미래 선택지를 좁힘**
- "아동 노동" 선택 순간 = 합리적 (인력 부족)
- 누적하면 = 독재자. 마지막 화면에서 "당신이 통과시킨 법률" 목록 → 자기 성찰
- **"이걸 선택해도 되는 건가?"** = Frostpunk의 핵심 감정

### 희망/불만 이중 게이지

- Hope(희망) 0 → 추방 (게임오버)
- Discontent(불만) 100 → 폭동 (게임오버)
- 대부분의 행동이 양쪽을 동시에 움직임 (비대칭적으로)
- 자연 감쇠하지만 위기 시 감쇠보다 빠르게 악화

### 자원과 "항상 출혈하는" 발전기

석탄이 항상 소모됨. 발전기 정지 = 주민 동사. 플레이어는 절대 안전하지 않고, "아직 위기가 아닐 뿐".

### 적용: 포고령을 비가역적 분기 트리로

현재 포고령(배급/훈련/경계) = 가역적 슬라이더 → 무게 없음.

**제안:**

```
통치 방식 트리:
  자비로운 군주 → 은혜 정치 → 자유 도시 (사기↑↑, 생산성↓)
  OR
  공포의 마왕 → 감시 체제 → 공포 제국 (이탈 불가, 폭주↑↑)

영웅 관리 트리:
  자율 방임 → 자유 원정 → 방랑 허용 (만족↑, 통제↓)
  OR
  엄격 통제 → 강제 배치 → 낙인 (효율↑, 사기↓)

식량 정책 트리:
  배급 → 긴축 배급 → 인신공양(영웅 희생 = 식량 대량)
  OR
  배급 → 풍족 배급 → 연회 문화 (사기↑, 자원↓↓)
```

각 법률이 **7대 죄악 점수에 기여** → 엔딩 분기 연결.
"나도 모르는 사이에 탐욕의 마왕이 되어 있었다."

### 적용: 거점 질서도 추가

사기 = 개별 영웅의 감정 (기존)
질서 = 거점 전체의 기강 (신규)

- 폭주 → 질서↓
- 포고령 → 질서↑ but 사기↓
- 질서 0 = 반란

## 6. 자원 관리의 재미

### 원칙 1: "항상 출혈하는 자원" 1개

Frostpunk 석탄 = 발전기가 항상 소모. 멈추면 사망.

우리 게임: 식량이 이 역할이지만 채집 1~2명이면 관리 가능 → 긴장감 부족.

**제안**: 식량 외에 **"거점 질서"가 매 턴 자연 하락** → 포고령/이벤트로 유지해야. 0 = 반란.

### 원칙 2: "하나만 선택" 강제 — 이벤트가 충돌을 만들어야

```
오늘 상인이 왔다 (교역에 영웅 필요)
  + 오늘 방어전 예고 (방어에 영웅 필요)
  + 분노 영웅이 "원정 보내달라" 요청
  + 건설 진행 중 (건설에 영웅 필요)

  7명으로 4개를 동시에 할 수 없다 → 뭘 포기?
```

### 원칙 3: "성장 = 위험" (RimWorld Wealth Tax)

거점 시설 수 + 자원 비축량에 비례하여 밤 습격 규모 상승.
"시설 다 짓고 자원 쌓으면 안전"이 아니라 **"성장할수록 습격이 거세다"**.

### 원칙 4: 자원에 "인격" 부여 (죄종 연동)

- 식량 부족 → 폭식 영웅 불만 + "배급 방식" 이벤트 강제
- 골드 과잉 → 탐욕 영웅 도난 확률↑ + 상인 방문 빈도↑
- 나무 부족 → 건설 불가 + "벌채지 분쟁" 이벤트

## 7. 전투를 메인 콘텐츠로

### 전투가 메인이 되려면 필요한 것

**a) 매 전투가 다르게 느껴져야 한다 — 적 유형 다양화**

```
현재: 적 3~5체, HP/ATK만 다름
개선: 적 유형별 행동 패턴이 다름
  [돌격형] → 직진 (기본)
  [궁수형] → 후방 원거리 공격 (접근해야 잡음)
  [방패형] → 앞에서 아군을 막음 (HP↑↑)
  [치유형] → 뒤에서 아군 회복 (먼저 잡아야 효율)
  [자폭형] → 달려와서 폭발 (범위 피해)
```

**b) 전투 전/중/후의 판단이 모두 의미 있어야**

```
전투 전:
  - 파티 편성 (누구를 보낼지)
  - 진형 선택 (어떤 전술로)
  - 죄종 시너지 고려

전투 중:
  - 바알 지시 (진격/후퇴/집중)
  - 죄종 기술 타이밍 (SP 관리)
  - 일기토 수락/거부
  - 증원 판단 (방어전)

전투 후:
  - 사기 변동 (승패, 일기토 결과, 동료 부상)
  - 계속 진행/철수 판단 (원정)
  - 전리품 분배 이벤트
```

**c) 전투 결과가 거점에 큰 파급**

- 영웅 A 일기토 승리 → 사기↑ → 거점 생산성↑
- 영웅 B 중상 → 병원 3턴 → 인력 부족 → 건설 지연
- 방어전 패배 → 시설 파손 + 자원 약탈 → 경제 위기

## 8. 연쇄 이벤트 설계

### 핵심 원칙: 다른 종류의 시스템을 경유해야 진짜 연쇄

```
현재 (같은 시스템 내 루프):
  폭주 → 사기 변동 → 폭주

RimWorld (시스템 경유):
  전투 → 부상 → 의료 → 작업 감소 → 식량 부족 → Mood↓ → Mental Break → 방화 → 시설 파괴 → 방어↓
```

### 체인 예시 1: "탐욕의 눈사태"

```

  ↓ [경제 시스템]
Day 8: 골드 부족 → 연구 중단, 영웅 고용 불가
  ↓ [이벤트 시스템]
Day 9: 아침 — "도둑을 처벌할 것인가?"
  선택지: 처벌(사기↓) / 용서(재발) / 전체 자원 공개(질서↑ 사기↓)
  ↓ [전투 시스템]
Day 9: 밤 습격 — 감시탑 정보 부족(연구 중단), 규모 모름
  ↓ [영웅 시스템]
Day 10: 방어전에서 영웅 부상 → 병원 필요
  ↓ [경제 시스템]
Day 10: 병원 미건설(골드 부족) → 자연 회복 3턴 → 인력 부족
  ↓ [사기 시스템]
Day 11: [분노] "인력 부족한데 왜 도둑을 안 벌했냐" → 사기↓
```

### 체인 예시 2: "전투의 여진"

```
Day 12: [교만] 일기토 도전 → 패배 → 중상 + 전체 사기↓
  ↓ [사기 시스템]
Day 12: [시기] "그가 져서 기쁘다" → 사기↑ (시기의 본질!)
  ↓ [이벤트 시스템]

  ↓ [관계 시스템]

  ↓ [거점 시스템]
Day 14: 같은 시설 배치 시 갈등 이벤트 확률↑
```

### 체인 예시 3: "긍정적 연쇄" (현재 기획에서 부족한 부분!)

```
Day 15: 방어전 대승 (피해 없이)
  ↓ [사기 시스템]
Day 15: 전체 사기 +10 → [분노] 고양 구간 진입
  ↓ [죄종 시스템]

  ↓ [원정 시스템]
Day 16: 높은 사기 → 죄종 기술 자동 발동 → 스테이지 클리어
  ↓ [경제 시스템]

```

### 4. 원정 실시간 전투

**방향**: 결과형(simulate 즉시완료) → 실시간(tick 단위 + 플레이어 개입)

**핵심**: 방어전(`MapDefenseMode`)이 이미 동일 패턴 구현 중 → 원정에 그대로 적용

**씬 플로우**:
```
현재: dispatch() → resolveExpedition() → simulate() 즉시완료 → ResultScene
개선: dispatch() → prepareExpeditionBattle() → BattleSceneA(realtime) → onClose() → finalizeExpedition() → ResultScene(간소화)
```

**변경 파일**:
- `src/game_logic/ExpeditionManager.js` — `resolveExpedition()`을 `prepareExpeditionBattle()` + `finalizeExpedition(result)`로 분리
- `src/scenes/MapScene.js` — `_processEveningPhase()`: 방어전 패턴 참고하여 BattleSceneA launch
- `src/scenes/ResultScene.js` — 전투 결과 제거, 보상/감시탑 정보만 남김

**카드 → 지시+죄종 기술 전환** (우선순위 P1~P2):

바알 지시 (쿨다운 재사용):
- 진격: ATK+30%/DEF-20% (쿨다운 5틱)
- 후퇴: DEF+30%/ATK-20%
- 집중공격: 지목 적에게 집중

죄종 기술 (사기 71+이면 자동, 31~70이면 수동):
- 분노: 광전사 / 교만: 불굴 / 시기: 저주 / 폭식: 포식 / 나태: 안식 / 색욕: 유혹 / 탐욕: 약탈

**검증 포인트**:
- BattleSceneA가 이미 realtime/replay 양 모드 지원 — 기술적 위험 최소
- 반복감 방지: 스킵 기능 유지 필수
- 구현 우선순위: P0 실시간 전환 → P1 지시 → P2 죄종 기술 → P3 일기토 선택
