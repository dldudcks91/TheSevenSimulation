# 이벤트/선택지 시스템 설계

> 상태: 이벤트 30개 초안 완성 + 조우(encounter) 이벤트 3개 (2026-04-16), 수치 밸런싱 미확정
> 작성일: 2026-03-24
> 마지막 업데이트: 2026-04-16

---

## 1. 개요

매 턴 1~2개 이벤트 발생. 플레이어의 판결 → 영웅 사기 변화 + 거점 상태 변화.
이벤트는 이 게임의 **심장**. 숫자 관리가 아니라 상황 대응.

## 1-1. 아침 보고 시스템

매 턴 아침, 이벤트 발생 전에 **영웅 심리 상태 보고 팝업**이 표시된다.
플레이어에게 "오늘 누구를 신경 써야 하는지" 알려주는 역할.

### 구조
- **위험 영웅** (사기 76+ / 25-): 서사적 한 줄 묘사로 개별 표시
- **안정 영웅**: "더보기" 버튼으로 접혀 있음, 펼치면 평상시 행동 한 줄
- **감시탑 정보**: 습격 예고가 있으면 하단에 표시

### 죄종별 서사 텍스트

| 죄종 | 안정 (26~75) | 고양/폭주 위험 (76+) | 불만/이탈 위험 (25-) |
|------|-------------|-------------------|-------------------|
| 분노 | 훈련장에서 허수아비를 박살내고 있습니다 | 이를 갈며 무기를 만지작거립니다 | 벽만 바라보며 아무 말도 하지 않습니다 |
| 시기 | 다른 영웅의 장비를 흘끔흘끔 쳐다봅니다 | 다른 영웅들을 노려보고 있습니다 | 자기 방에서 나오지 않습니다 |
| 탐욕 | 자기 소지품을 세고 또 세고 있습니다 | 창고 근처를 서성이고 있습니다 | 짐을 싸기 시작했습니다 |
| 나태 | 양지바른 곳에서 낮잠 중입니다 | 모두에게 '그만두자'고 말하고 다닙니다 | 며칠째 잠만 자고 있습니다 |
| 폭식 | 주방에서 뭔가 만들어 먹고 있습니다 | 식량을 혼자 끌어안고 있습니다 | 아무것도 먹지 않습니다 |
| 색욕 | 누군가에게 꽃을 꺾어주고 있습니다 | 특정 동료에게 집착하고 있습니다 | 아무도 만나려 하지 않습니다 |
| 교만 | 거울 앞에서 자세를 교정하고 있습니다 | 명령을 무시하기 시작했습니다 | 자존심이 무너져 침묵합니다 |

## 2. 이벤트 유형

| 유형 | 트리거 | 예시 |
|------|--------|------|
| 영웅 간 충돌 | 대립 죄종 영웅이 같은 시설 | [분노] vs [나태] 충돌 |
| 영웅 요청 | 죄종 불만 누적 | "원정 보내달라" |
| 폭주 대응 | 사기 100 | [탐욕]이 창고 털기 시도 |
| 외부 사건 | 턴/챕터 기반 | 상인 방문, 떠도는 영웅 |
| 밤 습격 후유증 | 밤 방어전 결과에 따라 | 부상자 발생, 시설 파손 |
| **조우 (encounter)** | 원정 중 event 노드 도달 | 부상당한 순례자, 낡은 제단, 버려진 보물상자 (2026-04-16 추가) |

### 조우 이벤트 (encounter) — 원정 전용

- **트리거**: `trigger_type='encounter'`, 아침 페이즈에서는 **발생하지 않음**
- **발생 방식**: ExpeditionScene에서 event 노드 클릭 → `EventSystem.pickEncounterEvent(partyHeroIds)` 호출
- **효과 범위**: `target='party'` — 원정 파티 영웅만 사기 변동 (`all`이 아님)
- **바알 죄종 변동**: `target='player_sin_<sin>'` — 예: `player_sin_lust` +2 (치료해주기 선택)
- **현재 풀**: F1(부상당한 순례자), F2(낡은 제단), F3(버려진 보물상자) — 각 3선택지

### Effect target 타입 (event_effects.csv)

| target | 의미 | 사용 필드 |
|--------|------|---------|
| `all` | 모든 영웅 사기 변동 | morale |
| `party` | 파티(원정 파견) 영웅만 (2026-04-16) | morale |
| `others` | 주인공 영웅 제외 | morale |
| `gold` | 골드 변동 | amount |
| `recruit` | 영입 액션 | action |
| `player_sin_<sin>` | 바알 죄종 수치 변동 (2026-04-16) | amount |
| `<sin>` (wrath 등) | 해당 죄종 최고 영웅 개별 | morale (음수 -999 = 제거) |
| `strongest`, `other`, `injured`, `heroA`, `heroB` | 특정 역할 영웅 | morale |

## 3. 설계 원칙

### 이벤트는 "사건"이다

> 나쁜 예: "[분노] 사기가 85에 도달했습니다"
> 좋은 예: "[분노]가 주점 벽에 주먹을 날겨 구멍을 냈습니다"

플레이어는 숫자가 아니라 **장면**을 본다.

### 선택지는 "누구 편이냐"를 묻는다

모든 선택지의 본질: **A를 살리면 B가 죽는다.**
- 가해자를 벌하면 → 가해자 불만, 피해자 안정
- 가해자를 감싸면 → 가해자 충성, 피해자 절망
- 무시하면 → 단기 비용 없음, 장기 무법지대

### 죄종의 본질이 드러나야 한다

| 죄종 | 이벤트에서 드러나는 본질 |
|------|----------------------|
| 분노 | **파괴** — 때리고, 부수고, 폭발한다 |
| 시기 | **비교** — 깎아내리고, 뒤에서 공작한다 |
| 탐욕 | **소유** — 훔치고, 독점하고, 나누지 않는다 |
| 나태 | **정지** — 안 나오고, 전염시키고, 무너뜨린다 |
| 폭식 | **과잉** — 먹고, 비축하고, 모자라게 만든다 |
| 색욕 | **집착** — 매달리고, 꼬시고, 삼각관계를 만든다 |
| 교만 | **군림** — 설교하고, 거부하고, 자기가 옳다 한다 |

### 기타 원칙

- 모든 선택지는 **강력하지만 대가가 있음**
- 정답이 없음 — 상황과 현재 상태에 따라 최선이 다름
- 선택 결과는 **영웅 사기에 직접 반영** (바알 수치 변동 없음)
- 선택지 3~4개. 수치는 추후 밸런싱

### 이벤트 구조 표준

```
[이벤트 제목]
트리거: (발생 조건)
장면: (무슨 일이 벌어졌는가 — 2~3문장)

▶ 선택지 A (행동 묘사)
  → 결과 묘사 + 사기 변동

▶ 선택지 B / C / D ...
```
