# 전투 시스템 설계

> 상태: **확정** — X축 오토배틀 + 죄종 반응 시스템
> 작성일: 2026-03-24
> 확정일: 2026-03-27 (죄종 반응 전환: 2026-04-13 / 전투씬 통합: 2026-04-15)

---

## 1. 개요

**X축 오토배틀 + 죄종 반응** — 영웅들이 자동으로 싸우고, 플레이어(바알)는 전투 중 죄종 반응 이벤트에 대해 판단(격려/제지)으로 개입한다.

### 핵심 컨셉
- 영웅은 **알아서 싸운다** (자동 이동, 자동 공격)
- 플레이어는 **바알 = 판결자** — 죄종 반응 이벤트에 대해 판단
- 전투 전: 파티 편성 + 작전 지시(공격적/방어적/균형)
- 전투 중: 죄종 반응 이벤트 대응 + 일기토 허가/금지
- 전투 중 죄종 기반 **일기토** 플레이어 허가제 (드라마 연출)

### 참고작
- **Crusaders Quest**: 횡스크롤 자동전투 + 스킬 블록
- **삼국지9/14**: 반자동 전투 — 지시 후 자동 진행
- **HoMM3**: 지휘관 버프, 파티 편성
- **Darkest Dungeon**: 로스터 관리, 영웅 상실 위험

## 2. 전투 화면

전투씬은 **원정 전투(BattleSceneA)** 와 **방어전(MapDefenseMode)** 두 가지를 포함한다.
SP 시스템, 죄종 반응, 일기토는 양쪽에서 동일하게 작동한다.

### 2-A. 원정 전투 (BattleSceneA — 별도 씬)

원정 파견 시 별도 씬으로 전환. 전투 종료 후 ResultScene으로 이동.

**작전 지시 팝업** (BattleSceneA 진입 전, MapScene 위 팝업):
```
┌─── 작전 지시 — 챕터1-3 봉화 초소 ─────────────────────┐
│  감시탑 정보: 오크 5~8명, 궁수 포함 (레벨2 감시탑)    │
│                                                       │
│  파티: [영웅A ★분노] [영웅B ★교만] [영웅C ★탐욕]    │
│  HP:   100/100       78/100        92/100            │
│                                                       │
│  작전: ○ 공격적  ● 균형  ○ 방어적                    │
│        (+15% ATK)  (기본)  (+15% DEF)                 │
│                                                       │
│                     [취소]   [출발 ▶]                 │
└───────────────────────────────────────────────────────┘
```

**전투 씬 레이아웃 (1280x720)**:

### 2-B. 방어전 (MapDefenseMode — MapScene 위 오버레이)

습격 발생 시 MapScene 위에 전투 오버레이. 별도 씬 전환 없음.
영외(우측) 에서만 전투 진행. 영내(좌측)는 딤 처리.

```
┌─── 헤더 (32px, 기존 MapScene HUD) ──────────────────────────────────┐
│  THE SEVEN │ Day 12 │ [밤] │ 💰500 │ 방어:4명                      │
├─── 영내 (딤) ────────────────┬─── 영외 전투 필드 ──────────────────┤
│                              │ [밤 습격 — 12일차]  R5              │
│  [건물들, 조작 불가]          │ SP ████░░ 42/100                   │
│                              │                                     │
│                              │ ★A→  ★B→  ★C→  ←○적1  ←○적2     │
│                              │                          ←○적3      │
│                              │                                     │
│                              │ [ ⚡ A: 광전사 ▲ ] [ ⚔ B: 일기토 ▲ ]│
├──────────────────────────────┴─────────────────────────────────────┤
│  전투 로그 (최근 5줄)                                               │
│  [증원 N명]                          [⏸정지]  [x1]  [스킵]        │
└─────────────────────────────────────────────────────────────────────┘
```

- 증원 가능 (대기 영웅 → 영외 필드 투입)
- 원정 전투와 동일: SP, 죄종 반응, 일기토 시스템 작동

### 시각 규격 (공통)
- 해상도: 1280x720
- 스프라이트: LPC 64x64, 런타임 합성 (SpriteRenderer)
- 방향: East(오른쪽)만 — 단일 행 스프라이트시트
- 액션: walk(이동/대기), slash(공격)

## 3. 전투 유형

| | 원정 전투 | 방어 전투 (밤 습격) | 사냥 |
|---|---|---|---|
| 편성 | 영웅 1~3명 | **방어 배치 영웅** | 영웅 1명 |
| 진행 | 별도 씬 (BattleSceneA) — 리플레이 가능 | **MapScene 오버레이 — 실시간 관전** | MapScene 위 팝업 (DuelBattleScene) |
| 죄종 반응 | 발동 (플레이어 선택) | 발동 (플레이어 선택) | 없음 |
| 증원 | 불가 (원정 중) | **가능** (대기 영웅 투입) | 불가 |

핵심 딜레마: 낮에 **영웅을 어떻게 배분할 것인가** — 원정에 보낸 영웅은 방어에 못 쓰고, HP가 깎인 영웅은 다음 원정·거점 행동이 제약된다.

## 4. 죄종 반응 시스템

플레이어(바알)는 **판결자**로서 전투 중 영웅의 죄종 반응에 개입한다.
경영 시뮬에 맞는 "사전 준비 85% + 전투 개입 15%" 구조.

### 전투 전: 작전 지시
- 원정/방어전 시작 전 **작전 지시** 선택 (공격적 / 방어적 / 균형)
- 지시에 따라 오토배틀 행동 패턴 변화

### 전투 중: 죄종 반응 이벤트
- 영웅 죄종 수치 조건 충족 시 **반응 트리거 패널에 표시** (예: 분노 쌓임 15+ 영웅 HP30%↓ → "광전사 돌입 ▲")
- 트리거 클릭 → **선택 팝업** 발동 (전투 일시정지)
  - `[격려]` — 효과 발동, 해당 죄종 +누적 / 부작용 가능
  - `[제지]` — 현상 유지, 다른 죄종 변동 가능 (TBD)
  - 8초 타임아웃 → 자동=제지
- 일기토도 동일 구조: `[허가]` / `[금지]`
- 매 전투마다 영웅 조합 × 죄종 쌓임 상태에 따라 다른 이벤트 → 반복 재미

### 죄종 반응 팝업 UI
```
┌─── ⚡ 죄종 반응 — 광전사 돌입 ──────────────────────┐
│                                                      │
│   [영웅A 스프라이트 — 분노 이펙트]                   │
│   "더 이상 참을 수 없다, 바알!"                      │
│   HP 30% 이하 · 분노 수치 16                         │
│                                                      │
│   ▸ [격려]  ATK+30%, 분노+2, 부상 위험↑              │
│   ▸ [제지]  현상 유지                                │
│                                                      │
│                              ⏱ 8초 (자동=제지)       │
└──────────────────────────────────────────────────────┘
```

## 5. 일기토 시스템

### 개요
전투 중 영웅의 **죄종 성격에 따라** 자동으로 적에게 일기토를 신청한다.
플레이어가 개입하지 않음 — 영웅 개인의 드라마 연출.

### 죄종별 일기토 성향

| 죄종 | 확률 | 대상 선택 |
|------|------|----------|
| 분노 | 50% | 랜덤 (싸우고 싶어 못 참음) |
| 교만 | 35% | 가장 강한 적 (자존심) |
| 시기 | 25% | 랜덤 (다른 영웅이 활약하면) |
| 탐욕 | 20% | 랜덤 |
| 폭식 | 15% | 가장 약한 적 (확실한 먹잇감) |
| 나태 | 5% | 거의 안 함 |
| 색욕 | 5% | 거의 안 함 |

### 적 수락/거부
- 수락 확률: 70%
- 거부 시: 적 ATK 소폭↓ + 신청 영웅 해당 죄종 +1 (예: 교만)

### 일기토 시각 연출
1. 오토배틀 일시정지
2. 화면 어둡게 + 중앙에 1:1 배치
3. 선제(민첩 비교) → 한 대씩 주고받기
4. 승패 후 오토배틀 복귀

### 결과 파급

| 결과 | 효과 |
|------|------|
| 아군 승리 | 적 ATK↓, 오토배틀 유리 + 신청 영웅 교만/분노 +1 |
| 아군 패배 | 영웅 부상/사망 + 파티원 분노 +1 → 2명으로 전투 |
| 적 거부 | 적 ATK 소폭↓ + 신청 영웅 교만 +1 |

> 상세 죄종 +수치는 정화/누적 카탈로그 확정 후 일괄 결정 (TBD).

## 6. 전투 구조

### X축 오토배틀
- 영웅(좌) vs 적(우), X축 이동만
- 가장 가까운 적을 향해 자동 이동
- 교전 거리 도달 → 자동 공격
- 근접/원거리 사거리 구분 (스프라이트별)

### 7스탯 전투 역할
- **힘**: 공격 데미지
- **민첩**: 행동 순서 + 이동 속도
- **건강**: 체력 게이지(1~100) 전용, 전투 미반영
- **통솔**: 방어전 ATK 보정
- **지능/감각/매력**: 전투 미반영 (거점 행동용)

### 공식 (balance.csv 기반)
```
영웅 HP = hero_hp_base(50) + 건강(vitality) × hero_hp_per_vitality(5)
매 턴 HP 자연 회복 = hero_hp_regen_per_turn(10)
ATK(원정) = 힘 × atk_expedition_str_mult(0.7) + 민첩 × atk_expedition_agi_mult(0.4)
ATK(방어) = 힘 × atk_defense_str_mult(0.8) + 통솔 × atk_defense_lead_mult(0.4)
데미지 = ATK × (0.8~1.2 랜덤) × 버프배율 × (1 - 피해경감) × 죄종 파워 보너스
```

### 죄종 파워 보너스 (쌓임 프레임 핵심 시스템)

각 죄종이 쌓일수록 전투 관련 효과가 커진다. 파워 보너스는 **해당 수치가 일정 구간에 도달했을 때만 활성화** — 플레이어가 "죄종을 어디까지 쌓을지" 선택하는 트레이드오프를 만든다.

| 죄종 | 전투 효과 (수치 쌓일수록 강화) | 구간 경계 |
|------|-----------------------------|---------|
| **분노** | ATK 보너스 | TBD |
| **시기** | SPD(민첩) 보너스 | TBD |
| **교만** | 선제 공격 보너스, 리더십 버프 | TBD |
| **탐욕** | 채집량/전리품 보너스 (전투 외) | TBD |
| **폭식** | HP 최대치/회복 보너스 | TBD |
| **색욕** | 파티 전원 ATK 소폭 버프 | TBD |
| **나태** | Stamina 소모 감소 (전투 외 유틸) | TBD |

**구간 설계(안)**:
- 0~? : 보너스 없음 (깨끗)
- ?~14 : 소폭 보너스
- 15~17 : 강화된 보너스 (고양 구간) — **고위험 고보상 영역**
- 18+ : 보너스는 최대지만 폭주 중 (문제 행동 동반)

> 구체 수치(구간 경계, 각 보너스 +N%)는 다음 세션에서 밸런싱.
> **딜레마**: "분노를 15~17에 유지하면 ATK +15%인데, 한 턴만 더 쌓이면 18 = 폭주 위험."

## 8. 영웅 HP 시스템

| 항목 | 값 |
|------|-----|
| 최대 HP | 50 + 건강(vitality) × 5 (기본 vitality 10 → HP 100) |
| 매 턴 자연 회복 | 10 (`hero_hp_regen_per_turn`) |
| 회복 가속 | 병원 시설(회복 턴 절반), 훈련장(+50%), 연금술소 치료약 |
| HP 0 | `knocked_out` → 귀환 시 `injured` (회복 턴 소요) |

- 영웅이 전투 피격 대상. 스프라이트 위에 HP 바 상시 표시
- 낮 거점 행동으로 깎인 **Stamina**와 분리 — HP는 순수 전투 내구, Stamina는 활동 자원

## 9. 증원 시스템

방어전에서 대기 영웅을 전투 중 투입 가능.

|------|------|
| 대상 | 거점 대기 중인 영웅 |

| 등장 | 필드 좌측 가장자리 |
| 제한 | 대기 영웅 수만큼 |

## 10. 밤 습격 (거점 방어전)

- **3~5일 간격**으로 습격 발생 (매일 밤 아님, balance.csv: raid_interval_min/max)
- 습격 없는 밤: 평화롭게 결산으로 넘어감
- **MapScene 위 오버레이**로 전투 진행 (별도 씬 전환 없음, MapDefenseMode)
- 규모: 소규모 정찰 ~ 대규모 공성 (day + 챕터 기반 스케일링)
- **방어: 낮에 "방어" 임무에 배치된 영웅만 참전** (자동 참여 아님)
- 방어 배치 0명 = 무방비 → 습격 자동 패배
- 감시탑 레벨에 따라 습격 정보 제공
- 방어 실패: 시설 피해, 자원 손실
