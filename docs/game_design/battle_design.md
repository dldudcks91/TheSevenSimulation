# 전투 시스템 설계

> 상태: **확정** — X축 오토배틀 + 죄종 반응 시스템
> 작성일: 2026-03-24
> 확정일: 2026-03-27 (죄종 반응 전환: 2026-04-13 / 전투씬 통합: 2026-04-15)

---

## 1. 개요

**X축 오토배틀 + 죄종 반응** — 영웅들이 자동으로 싸우고, 플레이어(바알)는 전투 중 죄종 반응 이벤트에 대해 판단(격려/제지)으로 개입한다.

### 핵심 컨셉
- 영웅은 **알아서 싸운다** (자동 이동, 자동 공격)
- 플레이어는 **바알 = 판결자** — 죄종 반응 이벤트에 대해 판단
- 전투 전: 파티 편성 + 작전 지시(공격적/방어적/균형)
- 전투 중: 죄종 반응 이벤트 대응 + 일기토 허가/금지
- 전투 중 죄종 기반 **일기토** 플레이어 허가제 (드라마 연출)

### 참고작
- **Crusaders Quest**: 횡스크롤 자동전투 + 스킬 블록
- **삼국지9/14**: 반자동 전투 — 지시 후 자동 진행
- **HoMM3**: 지휘관 버프, 파티 편성
- **Darkest Dungeon**: 로스터 관리, 영웅 상실 위험

## 2. 전투 화면

전투씬은 **원정 전투(BattleSceneA)** 와 **방어전(MapDefenseMode)** 두 가지를 포함한다.
SP 시스템, 죄종 반응, 일기토는 양쪽에서 동일하게 작동한다.

### 2-A. 원정 전투 (BattleSceneA — 별도 씬)

원정 파견 시 별도 씬으로 전환. 전투 종료 후 ResultScene으로 이동.

**작전 지시 팝업** (BattleSceneA 진입 전, MapScene 위 팝업):
```
┌─── 작전 지시 — 챕터1-3 봉화 초소 ─────────────────────┐
│  감시탑 정보: 민병 15~20명, 궁수 포함 (레벨2 감시탑)   │
│                                                       │
│  파티: [영웅A ★분노] [영웅B ★교만] [영웅C ★탐욕]    │
│  병사: ──────O────── 8명     (방어 잔여: 12명)         │
│                                                       │
│  작전: ○ 공격적  ● 균형  ○ 방어적                    │
│        (+15% ATK)  (기본)  (+15% DEF)                 │
│                                                       │
│                     [취소]   [출발 ▶]                 │
└───────────────────────────────────────────────────────┘
```

**전투 씬 레이아웃 (1280x720)**:
```
┌─── 헤더 (56px) ─────────────────────────────────────────────────────┐
│  챕터1-3 봉화 초소   민병: 8/10   R12   SP ████████░░░░ 42/100     │
├─── 전장 (440px) ────────────────────────────────────────────────────┤
│                                                                     │
│  [영웅A]  [영웅B]  [영웅C]     ←→     [적1]  [적2]  [적3]         │
│  HP████   HP████   HP██░░              HP██░░  HP████               │
│  (죄종색, 비율별 컬러)                   (적색)                       │
│                                                                     │
├─── 죄종 반응 트리거 (64px) ─────────────────────────────────────────┤
│  [ ⚡ A: 광전사 돌입 ▲ ]  [ ⚔ A↔적1: 일기토 ▲ ]  [ — ]  [ — ]   │
```

### 2-B. 방어전 (MapDefenseMode — MapScene 위 오버레이)

습격 발생 시 MapScene 위에 전투 오버레이. 별도 씬 전환 없음.
영외(우측) 에서만 전투 진행. 영내(좌측)는 딤 처리.

```
┌─── 헤더 (32px, 기존 MapScene HUD) ──────────────────────────────────┐
│  THE SEVEN │ Day 12 │ [밤] │ 💰500 │ 방어:4명                      │
├─── 영내 (딤) ────────────────┬─── 영외 전투 필드 ──────────────────┤
│                              │ [밤 습격 — 12일차]  민병: 8/10  R5  │
│  [건물들, 조작 불가]          │ SP ████░░ 42/100                   │
│                              │                                     │
│                              │ ★A→  ★B→  ★C→  ←○적1  ←○적2     │
│                              │                          ←○적3      │
│                              │                                     │
│                              │ [ ⚡ A: 광전사 ▲ ] [ ⚔ B: 일기토 ▲ ]│
├──────────────────────────────┴─────────────────────────────────────┤
│  전투 로그 (최근 5줄)                                               │
│  [증원 N명]                          [⏸정지]  [x1]  [스킵]        │
└─────────────────────────────────────────────────────────────────────┘
```

- 증원 가능 (대기 영웅 → 영외 필드 투입)
- 원정 전투와 동일: SP, 죄종 반응, 일기토 시스템 작동

### 시각 규격 (공통)
- 해상도: 1280x720
- 스프라이트: LPC 64x64, 런타임 합성 (SpriteRenderer)
- 방향: East(오른쪽)만 — 단일 행 스프라이트시트
- 액션: walk(이동/대기), slash(공격)
- 병사: 스프라이트 없음, 카운터 텍스트로 표시

## 3. 전투 유형

| | 원정 전투 | 방어 전투 (밤 습격) | 사냥 |
|---|---|---|---|
| 편성 | 영웅 1~3명 + 병사 N명 | **방어 배치 영웅 + 잔여 병사** | 영웅 1명 (병사 없음) |
| 진행 | 별도 씬 (BattleSceneA) — 리플레이 가능 | **MapScene 오버레이 — 실시간 관전** | MapScene 위 팝업 (DuelBattleScene) |
| 죄종 반응 | 발동 (플레이어 선택) | 발동 (플레이어 선택) | 없음 |
| 증원 | 불가 (원정 중) | **가능** (대기 영웅 투입) | 불가 |

핵심 딜레마: 낮에 **영웅 + 병사를 어떻게 배분할 것인가** — 원정에 영웅+병사를 많이 보내면 방어가 약해지고, 방어에 남기면 원정 진행이 느려짐.

## 4. 죄종 반응 시스템

플레이어(바알)는 **판결자**로서 전투 중 영웅의 죄종 반응에 개입한다.
경영 시뮬에 맞는 "사전 준비 85% + 전투 개입 15%" 구조.

### 전투 전: 작전 지시
- 원정/방어전 시작 전 **작전 지시** 선택 (공격적 / 방어적 / 균형)
- 지시에 따라 오토배틀 행동 패턴 변화

### 전투 중: 죄종 반응 이벤트
- 영웅 죄종 조건 충족 시 **반응 트리거 패널에 표시** (예: 분노 높은 영웅 HP30%↓ → "광전사 돌입 ▲")
- 트리거 클릭 → **선택 팝업** 발동 (전투 일시정지)
  - `[격려]` — 효과 발동, 사기↑ / 부작용 가능
  - `[제지]` — 현상 유지, 사기↓
  - 8초 타임아웃 → 자동=제지
- 일기토도 동일 구조: `[허가]` / `[금지]`
- 매 전투마다 영웅 조합 × 죄종 상태에 따라 다른 이벤트 → 반복 재미

### 죄종 반응 팝업 UI
```
┌─── ⚡ 죄종 반응 — 광전사 돌입 ──────────────────────┐
│                                                      │
│   [영웅A 스프라이트 — 분노 이펙트]                   │
│   "더 이상 참을 수 없다, 바알!"                      │
│   HP 30% 이하 · 분노 수치 72                         │
│                                                      │
│   ▸ [격려]  ATK+30%, 사기↑8, 부상 위험↑              │
│   ▸ [제지]  현상 유지, 사기↓5                        │
│                                                      │
│                              ⏱ 8초 (자동=제지)       │
└──────────────────────────────────────────────────────┘
```

## 5. 일기토 시스템

### 개요
전투 중 영웅의 **죄종 성격에 따라** 자동으로 적에게 일기토를 신청한다.
플레이어가 개입하지 않음 — 영웅 개인의 드라마 연출.

### 죄종별 일기토 성향

| 죄종 | 확률 | 대상 선택 |
|------|------|----------|
| 분노 | 50% | 랜덤 (싸우고 싶어 못 참음) |
| 교만 | 35% | 가장 강한 적 (자존심) |
| 시기 | 25% | 랜덤 (다른 영웅이 활약하면) |
| 탐욕 | 20% | 랜덤 |
| 폭식 | 15% | 가장 약한 적 (확실한 먹잇감) |
| 나태 | 5% | 거의 안 함 |
| 색욕 | 5% | 거의 안 함 |

### 적 수락/거부
- 수락 확률: 70%
- 거부 시: 적 전체 사기↓ + 신청 영웅 사기↑

### 일기토 시각 연출
1. 오토배틀 일시정지
2. 화면 어둡게 + 중앙에 1:1 배치
3. 선제(민첩 비교) → 한 대씩 주고받기
4. 승패 후 오토배틀 복귀

### 결과 파급

| 결과 | 효과 |
|------|------|
| 아군 승리 | 적 전체 사기↓, 오토배틀 유리 + 신청 영웅 사기↑ |
| 아군 패배 | 아군 사기↓ + 영웅 부상/사망 → 2명으로 전투 |
| 적 거부 | 적 사기↓ (소량) + 신청 영웅 사기↑ |

## 6. 전투 구조

### X축 오토배틀
- 영웅(좌) vs 적(우), X축 이동만
- 가장 가까운 적을 향해 자동 이동
- 교전 거리 도달 → 자동 공격
- 근접/원거리 사거리 구분 (스프라이트별)

### 7스탯 전투 역할
- **힘**: 공격 데미지
- **민첩**: 행동 순서 + 이동 속도
- **건강**: 체력 게이지(1~100) 전용, 전투 미반영
- **통솔**: 방어전 ATK 보정
- **지능/감각/매력**: 전투 미반영 (거점 행동용)

### 공식 (balance.csv 기반)
```
영웅 HP 없음 — 병사가 피격 담당 (병사 HP = soldier_hp(80))
ATK(원정) = 힘 × atk_expedition_str_mult(0.7) + 민첩 × atk_expedition_agi_mult(0.4)
ATK(방어) = 힘 × atk_defense_str_mult(0.8) + 통솔 × atk_defense_lead_mult(0.4)
데미지 = ATK × (0.8~1.2 랜덤) × 버프배율 × (1 - 피해경감)
```

## 7. 사망 시스템

### 영웅

| 상태 | 조건 | 효과 |
|------|------|------|
| 기절 | HP 0 | 귀환 후 회복 (1~2턴) |
| 중상 | 기절 + 추가 피격 | 장기 치료 (3~5턴) |
| 사망 | 중상 + 추가 피격 | **영구 상실** |

### 병사
- HP 0 → **즉시 사망, 영구 소실**
- 병사는 소모품. 전투 후 생존 병사만 풀로 복귀

---

## 8. 병사 시스템

| 항목 | 값 |
|------|-----|
| 종류 | 민병 (1종) |
| HP | 80 |
| ATK | 8 |
| SPD | 3 |
| 시작 시 | 100명 |
| 모집 | 골드로 (비용 후일 기획) |

- 화면에 스프라이트 없음 (카운터 표시)
- 적은 병사를 우선 공격 → 병사 전멸 시 영웅이 직접 맞음
- 원정 배분 vs 방어 잔여 = 핵심 딜레마

## 9. 증원 시스템

방어전에서 대기 영웅을 전투 중 투입 가능.

| 항목 | 내용 |
|------|------|
| 대상 | 거점 대기 중인 영웅 |
| 비용 | 없음 (다음 턴 피로 페널티) |
| 등장 | 필드 좌측 가장자리 |
| 제한 | 대기 영웅 수만큼 |

---

## 10. 밤 습격 (거점 방어전)

- **3~5일 간격**으로 습격 발생 (매일 밤 아님, balance.csv: raid_interval_min/max)
- 습격 없는 밤: 평화롭게 결산으로 넘어감
- **MapScene 위 오버레이**로 전투 진행 (별도 씬 전환 없음, MapDefenseMode)
- 규모: 소규모 정찰 ~ 대규모 공성 (day + 챕터 기반 스케일링)
- **방어: 낮에 "방어" 임무에 배치된 영웅만 참전** (자동 참여 아님)
- 방어 배치 0명 = 무방비 → 습격 자동 패배
- 감시탑 레벨에 따라 습격 정보 제공
- 방어 실패: 시설 피해, 자원 손실
