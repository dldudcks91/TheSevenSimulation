# 전투 시스템 설계

> 상태: TheSevenTactics 계승, 시뮬레이션 맥락으로 조정
> 작성일: 2026-03-24

---

## 1. 개요

**하이브리드 자동전투** — 전투 자체는 자동 진행되지만, 플레이어는 전투 중 **증원** 판단이 가능. 전투 전(편성, 장비, **병사 배분**)과 전투 중(증원 투입), 전투 후(계속/철수)에 판단이 집중.

### 참고작
- **삼국지6**: 꿈 시스템 — 영웅의 성격/야망이 행동에 영향
- **삼국지9/14**: 반자동 전투 — 지시 후 자동 진행, 중간 개입 가능
- **HoMM3**: 지휘관 + 유닛 편성
- **Darkest Dungeon**: 로스터 관리, 위치 기반 전투

### 전투 방식 (A/B안 병행)

| | A안: 돌진형 (BattleSceneA) | B안: 필드 이동형 (BattleSceneB) ← **현재 기본값** |
|---|---|---|
| 컨셉 | 카드 배치 후 완전 자동 | 유닛 자유 이동, 증원 가능 |
| 배치 | 전투 전 카드 배치식 편성 | 초기 배치 후 자유 이동 |
| 조작 | 없음 (완전 자동) | 증원 투입 (대기 영웅 선택 → 필드 가장자리에서 등장) |
| 연출 | 돌진 → 충돌 → 결과 | 필드 위 실시간 이동/교전 |
| 적합 | 빠른 전투 소화, 원정 결과 | 방어전, 보스전 등 드라마틱 전투 |

### 전투 유형

| | 원정 전투 | 방어 전투 (밤 습격) | 사냥 |
|---|---|---|---|
| 편성 | 영웅 1~3명 + 병사 N명 | 잔류 영웅 + 잔여 병사 | 영웅 1명 (병사 없음) |
| 진행 | 파견 후 결과만 받음 | **실시간으로 지켜봄** (증원 가능) | 실시간 1:1 전투 |
| 조작 | 없음 (완전 자동) | 증원 투입 가능 | 없음 (완전 자동) |
| 전투 방식 | B안 (필드 이동형) | B안 (필드 이동형) | B안 (필드 이동형) |

핵심 딜레마: 원정에 보낸 **영웅 + 병사**는 밤 방어에 못 씀. + 증원을 언제 투입할 것인가?

## 2. 전투 구조

- 7스탯 기반 자동 계산
  - 원정 전투: 힘(공격), 민첩(행동순서), 체력(생존), 지능(AI 판단)
  - 밤 방어: 힘(공격), 통솔(팀 버프), 체력(생존)
- 명중/회피 없음, 마나 없음
- A안: 배치 후 완전 자동 / B안: 자동 진행 + 증원 개입

## 3. 사망 시스템 (TheSevenTactics 계승)

### 영웅

| 상태 | 조건 | 효과 |
|------|------|------|
| 기절 | HP 0 | 귀환 후 회복 (1~2턴) |
| 중상 | 기절 + 추가 피격 | 장기 치료 (3~5턴) |
| 사망 | 중상 + 추가 피격 | **영구 상실** |

### 병사

- HP 0 → **즉시 사망, 영구 소실**
- 병사는 소모품. 전투 후 생존 병사만 풀로 복귀

---

## 4. 병사 시스템

### 개요

거점에 **병사 풀**이 존재. 영웅 출격 시 병사를 배정하여 함께 전투.
병사는 이름 없는 소모성 유닛 — 영웅의 방패막이이자 전투력 보조.

### 병사 기본 스펙

| 항목 | 값 | 비고 |
|------|-----|------|
| 종류 | **민병** (1종) | 나중에 궁수/기사 등 확장 가능 |
| HP | 30 | 고정 |
| ATK | 5 | 고정 |
| SPD | 3 | 고정 |
| 개별 스탯 | 없음 | 모든 병사 동일 |
| 성장 | 없음 | |

### 병사 풀

```
거점 병사 풀: N명
  │
  ├── 원정 영웅 A에게 10명 배정 → 풀에서 차감
  ├── 원정 영웅 B에게 5명 배정 → 풀에서 차감
  └── 잔여 → 밤 방어에 자동 투입
```

- 프로토타입(Phase 1): **시작 시 100명**
- 전투에서 죽은 병사는 영구 소실
- 생존 병사는 전투 후 풀로 복귀
- 모집: 골드로 모집 (비용/방식 후일 기획)
- 통솔 스탯: 병사 **모집** 시 영향 (효율/품질 — 후일 기획)

### 출격 규칙

| 규칙 | 내용 |
|------|------|
| 영웅 없이 출격 | 불가 |
| **병사 없이 출격** | **불가** (최소 1명 필요) |
| 사냥 | 예외 — 영웅 1명 단독 (병사 불필요) |

### 전투에서의 행동

- 병사는 **개별 유닛**으로 전투에 참여 (BattleEngine에서 1체씩 처리)
- **적은 병사만 공격 가능** — 영웅은 타겟 불가 (영웅은 지휘관 위치)
- 영웅은 적을 공격하지만, 적은 영웅을 공격할 수 없음
- **병사 전멸 → 영웅 이탈 (전투 패배)** — 영웅은 부상 없이 안전하게 후퇴
- 병사의 공격은 약하지만 **숫자로 압도**

### 배분 딜레마

```
병사 풀: 30명
  ├── 원정에 20명 배정 → 원정 안전, but...
  └── 밤 방어 남은 병사: 10명 → 방어 취약!

병사 풀: 30명
  ├── 원정에 5명 배정 → 원정 위험
  └── 밤 방어 남은 병사: 25명 → 방어 안전
```

> 기존 "영웅을 어디에 배치할까" 딜레마에 + "병사를 어떻게 나눌까" 딜레마 추가

## 5. 증원 시스템 (B안 전용)

### 개요

B안(필드 이동형)에서는 전투 중 **대기 영웅을 증원**으로 투입할 수 있다. 증원은 플레이어가 전투 상황을 보고 판단하는 **유일한 실시간 개입** 수단.

### 증원 흐름

```
전투 진행 중
  │
  ├── 화면 하단: 대기 영웅 목록 표시
  │     (거점에 남아있고 출격하지 않은 영웅)
  │
  ├── 플레이어가 대기 영웅 선택
  │     └── 해당 영웅이 **필드 가장자리**에서 등장
  │           (등장 위치: 아군 진영 방향 가장자리)
  │
  └── 증원 영웅은 이후 자동으로 전투 참여
```

### 규칙

| 항목 | 내용 |
|------|------|
| 증원 대상 | 거점에 대기 중인 영웅 (원정 파견/부상 제외) |
| 증원 비용 | 없음 (단, 증원한 영웅은 다음 턴 피로 페널티) |
| 등장 위치 | 필드 가장자리 (아군 진영 방향) |
| 등장 후 행동 | 자동 전투 AI에 따라 행동 |
| 증원 제한 | 전투당 제한 없음 (대기 영웅 수만큼 가능) |

### 전략적 의미

- 병력 온존: 처음부터 전력 투입 vs 여유분 남기기
- 타이밍 판단: 너무 늦으면 이미 병사 전멸, 너무 빠르면 다음 전투에 인력 부족
- 밤 방어에서 특히 중요: 습격 규모 확인 후 증원 여부 결정

## 6. 전투 시각화

### 렌더링

- **Phaser.js** 기반, tick 단위 실시간 렌더링
- A안/B안 모두 동일한 렌더링 파이프라인 사용
- 전투 로직(BattleEngine)은 tick 단위로 결과를 산출, 씬이 시각화

### LPC 스프라이트

| 항목 | 스펙 |
|------|------|
| 크기 | **64x64** px |
| 방향 | **4방향** (상/하/좌/우) |
| 액션 | **idle / walk / slash / hurt** |
| 포맷 | 스프라이트시트 (LPC Universal Sprite Sheet 규격) |

### 액션별 용도

| 액션 | 용도 | 비고 |
|------|------|------|
| idle | 대기, 전투 시작 전 | 기본 상태 |
| walk | 이동 (B안), 돌진 (A안) | 4방향 |
| slash | 근접 공격 | 공격 모션 |
| hurt | 피격 | 피격 리액션 |

## 7. 밤 습격 (거점 방어전)

- **매일 밤** 적 습격 발생
- 규모 가변: 소규모 정찰 ~ 대규모 공성 (챕터 진행에 따라 강화)
- 거점에 남은 **영웅 + 잔여 병사**로 자동 방어
- 영웅 없이 병사만으로도 방어 시도 가능 (약하지만)
- **감시탑** 업그레이드 → 습격 규모/적 구성 사전 정보 제공
- 방어 실패 시: 시설 피해, 자원 손실
