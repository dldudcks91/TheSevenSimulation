## 1. 바알(플레이어)의 역할 — 보존

바알은 **판결자이자 영주**다. 바알 자신의 유동 수치는 없고, 대신 **국시(國是, Edict)**를 통해 통치 방향을 직접 결정한다:

- 7죄종 중 1개를 **국시로 선포** (기간형, 지속 턴 = `[balance.csv:edict_duration]`)
- 선포한 국시의 효과가 거점 전반에 적용 (전투력/자원/생산성/영웅 사기 변동)
- 만료 후 공백 기간(`[balance.csv:edict_cooldown_normal]` 턴)을 거쳐 다음 국시 선포 가능
- 이벤트 선택지는 영웅들의 **사기 modifier**에 영향 (이전: 죄종 수치)
- 상세 규칙: §5 국시(Edict) 시스템

## 2. 7대 죄악 = 영웅의 메인 죄종 (정체성)

> **2026-04-27 전면 개편**: 7수치 누적 프레임 폐기 → 메인 죄종 1개 고정 + 사기 1~100.

### 핵심 모델

| 차원 | 담당 | 변동성 |
|---|---|---|
| **메인 죄종** | 영웅의 정체성 / 폭주 양상 결정 | **불변** (이름과 1:1 고정) |
| **시작특성** | 능력/사기/폭주 양상 변형 | **불변** (이름과 1:1 고정) |
| **사기 (Morale)** | 영웅의 현재 정신 상태 | **동적 변동** (1~100) |

→ "이 영웅은 분노 영웅이다" (불변) + "이 영웅의 사기가 지금 32다" (동적) = 직교.

### 7대 죄악 = 7가지 메인 죄종

| 메인 죄종 | 본질 | 폭주 시 양상 |
|------|---------|---|
| **분노** | 화 / 적개심 | Berserk — 아군 공격, 적 추격 강박 |
| **교만** | 우월감 / 자존감 비대 | 명령 거부, 단독 행동 |
| **탐욕** | 갈망 / 소유욕 | 자원 횡령, 창고 약탈 |
| **시기** | 질투 / 박탈감 | 사보타주, 타 영웅 약화 시도 |
| **색욕** | 집착 / 갈구 | 배치 거부, 특정 영웅 집착 |
| **폭식** | 결핍 / 공허 | 식량 소진, 폭식 |
| **나태** | 무기력 / 의욕 상실 | 행동 거부, 무기력 전파 |

> 폭주 양상은 **메인 죄종 × 시작특성**으로 변주 (147가지). 자세한 변형은 [hero_design.md §2-2](hero_design.md#2-2-사기morale-시스템--rimworld-mood-방식) 참조.

### 7죄종 × 7영역 × 7스탯 삼각 매핑 — 의미 재정의

이전엔 "고양 구간 스탯 보너스"의 근거였으나, 메인 죄종 고정 전환 후엔 **"메인 죄종이 정체성으로서 강조하는 영역"**으로 의미 재정의.

| 메인 죄종 | 영역 (도메인) | 관련 스탯 | 주 게임 활동 |
|---|---|---|---|
| **분노** | 🗡️ 전투 | 힘 | 원정 전투, 사냥, 방어전 |
| **교만** | 👑 명성·지휘 | 통솔 | 파티 리더, 외교, 위계 |
| **탐욕** | 💰 자원·경제 | 지능 | 채집, 벌목, 거래, 전리품 |
| **시기** | 👁️ 정찰·정보 | 민첩 | 정찰, 약점 간파, 비밀 수집 |
| **색욕** | 💋 사회·관계 | 매력 | 주점, 고용, 관계 형성 |
| **폭식** | 🍖 식량·체력 | 건강 | 농장, 식량 조달, HP |
| **나태** | 🛌 회복·관조 | 감각 | 숙소, Stamina 회복, 지속력 |

→ 메인 죄종은 영웅이 **어느 영역에서 활약하기 좋은가**의 힌트로 작동 (소프트 가이드, 강제력 없음).

### 사기 시스템 — 본격 설명은 hero_design.md로

- 단일 게이지 1~100, 기본값 50
- modifier 합산식 (RimWorld Mood/Thought 방식)
- 폭주 임계(낮은 사기) / 구원 임계(높은 사기)
- Catharsis 안전장치
- 자세한 시스템·modifier 카탈로그·확률 곡선: [hero_design.md §2-2](hero_design.md#2-2-사기morale-시스템--rimworld-mood-방식) §4

## 2-A. 폐기된 시스템 (2026-04-27)

이력 보존을 위해 명시. 모든 항목은 본 개편으로 **삭제**됨.

| 폐기 항목 | 대체 |
|---|---|
| `hero.sinStats` 7수치 누적 | `hero.morale` 1수치 + `hero.modifiers[]` |
| `topSin()` 동적 계산 | `hero.mainSin` 정적 필드 |
| `hero.sinPriority` 숨은 우선순위 순열 | 불필요 (메인 정적이라 동률 없음) |
| 4구간 시스템 (깨끗/발현/고양/폭주/임계) | 사기 임계 (불안/정상/고양/구원/폭주) |
| 발현 +1 / 고양 +2 / 폭주 +3 스탯 보너스 | 사기 고양 시 능력 보너스 / 불안 시 페널티 |
| 정화 수단 카탈로그 (죄종별 7종) | 사기 회복 (정화 = 사기 +offset modifier) |
| 임계 이벤트 (4구간 도달 시) | 사기 폭주/구원 영역에서 확률 발동 |
| 수식어 시스템 (`hero_epithets.csv`) | **폐기** (이름 단독 표시) |
| 시작특성 편향/저항/중립 분류 | 폐기 — 메인 고정이라 편향 무의미. 시작특성은 능력+사기+폭주 양상에 다축 영향 |

코드/CSV 마이그레이션 계획은 [_migration_findings.md](../_migration_findings.md) 참조.

## 2-B. 영웅 관계 (bonds) — 매트릭스 적용 방식 변경

### 구조 (보존)

- `hero.bonds = { [otherHeroId]: bonds_min..bonds_max }` — 영웅간 친밀도
- **Asymmetric** — A→B와 B→A가 다를 수 있음
- 초기값: `[balance.csv:bonds_initial]` (중립)
- **바알↔영웅 친밀도는 별도 축으로 두지 않음** — 후천 특성으로 이산 기록

### 수치 구간 해석 (보존)

| 구간 | 의미 |
|---|---|
| 하위 (~ `bonds_low_threshold` 부근) | 증오/원수 / 불신·껄끄러움 |
| 중립대 | 평범한 동료 |
| 상위 (`bonds_high_threshold` 이상) | 호감 / 절친·혈맹 |

### 성향 판정 — 메인 죄종 기반으로 단순화 (2026-04-27 변경)

이전: `topSin` 동적 + `sinPriority` 동률 해체 → **폐기**.
이후: **두 영웅의 메인 죄종**을 §3 매트릭스에 직접 대입 → 배율 결정.

```
A → B bonds 변동:
  관계 = matrix[A.mainSin][B.mainSin]
  배율 = matrix_multiplier[관계]  // 동류 ×2 / 강화 ×1.5 / 중립 ×1 / 대립 ×-2
```

→ `sinPriority` 필요 없음. 메인 고정이라 동률 없음. 매번 같은 결과.

### bonds 변동 배율 계산 순서 (2026-04-27 갱신)

```
1. 기본 Δ 산출 (사건별 수치)
2. 긍정 사건이면 × 관계배율 (동류 ×2 / 강화 ×1.5 / 중립 ×1 / 대립 ×-2)
3. × 색욕 영웅 배율 (주체 메인 죄종 = 색욕일 때, [balance.csv:bonds_lust_main_mult])
4. × 열정령 국시 배율 (국시 활성 시)
5. Math.round
6. 분노 가드 (주체 메인 죄종 = 분노 && Δ>0 → Δ=0, 또는 사기 폭주 영역일 때)
7. bonds clamp [`bonds_min`, `bonds_max`]
```

> 이전의 "고양 구간일 때만 배율 적용"은 폐기. 메인 죄종 기반으로 영구 적용. 단, 사기가 폭주 영역인 경우 분노 가드 등 발동.

### 매트릭스 적용 범위 (보존)

| 사건 카테고리 | 매트릭스 적용? | 근거 |
|---|:-:|---|
| **긍정 사건** (공유 경험, 구조, 선행) | ✅ 적용 | 같은 경험도 메인 죄종이 다르면 반응이 다름 |
| **부정 사건** (폭주 피해, 배신, 횡령) | ❌ 미적용 | 배신은 누구에게나 배신. 원본 Δ 그대로 |
| **상시 특성 배경** (구원 특성 등 /턴) | ❌ 미적용 | 특성 자체가 성격 표현 — 이중 적용 방지 |

### 관계 → 사기 자동 반영 (이전: 죄종 쌓임)

관계 변화는 별도 "감정 축"을 만들지 않고 **사기 modifier 추가**로 반영된다.

| 사건 | 관계 기준 | 사기 modifier |
|---|---|---|
| B가 죽음 | A.bonds[B] ≥ `bonds_high_threshold` | A에게 "동료 사망" -25 (20턴) |
| B가 죽음 | A.bonds[B] ≤ `bonds_low_threshold` | A에게 영향 없음 (또는 +5 — 기뻐함) |
| B가 원정 성공 | A.bonds[B] ≤ `bonds_low_threshold` | A에게 "타인의 영광" -3 (3턴) |
| 같은 시설 + A.bonds[B] 낮음 | — | A·B 모두 "껄끄러운 동거" -2 (situational) |
| 같은 시설 + A.bonds[B] 높음 | — | A·B 모두 "친구와 함께" +3 (situational) |
| B의 폭주가 C 공격 | C.bonds[B] > 중립대 | C에게 "친구의 배신" -10 (10턴) |

> 사건별 modifier 수치는 `balance.csv` `morale_mod_*` 키로 이전 예정.

## 3. 죄종 간 관계 매트릭스

> bonds 배율 계산의 기반이 되는 7×7 관계 표.
> **메인 죄종 고정**으로 매트릭스 적용이 단순해짐 — 두 영웅의 (고정) 메인끼리 비교.
>
> ⚠️ 현재 `sin_relations.csv`는 옛 설계 상태. 본 매트릭스는 CSV 갱신 전까지 **임시 SSOT** — `_migration_findings.md` F-A1 참조.

### 4가지 관계 유형

| 관계 | 배율 | 의미 |
|---|:-:|---|
| **동류** (같은 메인) | **×2** | "같은 죄를 짊어진 자끼리" — 가장 강한 결속 |
| **강화** | **×1.5** | 한 짝의 성향 궁합 — 서로를 북돋움 |
| **중립** | **×1** | 특별한 상호작용 없음 |
| **대립** | **×-2** | 한 짝의 성향 충돌 — 부호 반전 |

### 매트릭스 (보존)

|  | 교만 | 시기 | 분노 | 나태 | 탐욕 | 폭식 | 색욕 |
|:-:|:-:|:-:|:-:|:-:|:-:|:-:|:-:|
| **교만** | **동류** | **대립** | **대립** | **대립** | **대립** | **대립** | **대립** |
| **시기** | 대립 | **동류** | 강화 | · | · | · | 대립 |
| **분노** | 대립 | 강화 | **동류** | 대립 | · | · | · |
| **나태** | 대립 | · | 대립 | **동류** | · | 강화 | · |
| **탐욕** | 대립 | · | · | · | **동류** | 대립 | 강화 |
| **폭식** | 대립 | · | · | 강화 | 대립 | **동류** | · |
| **색욕** | 대립 | 대립 | · | · | 강화 | · | **동류** |

### 페어 구성 요약

- **동류 ++** 7쌍 (대각선, 같은 메인 죄종끼리)
- **강화** 3쌍: 분노↔시기 / 탐욕↔색욕 / 폭식↔나태
- **대립** 9쌍:
  - 교만 → 타 6개 죄종 모두 (6쌍) — 교만 완전 고립
  - 분노↔나태 / 탐욕↔폭식 / 시기↔색욕 (3쌍)

### 죄종별 관계 분포

| 메인 죄종 | 강화 | 대립 | 중립 | 특징 |
|---|:-:|:-:|:-:|---|
| **교만** | 0 | 6 | 0 | **자기밖에 모른다** — 모든 타 죄종과 대립 |
| **시기** | 1 (분노) | 2 (교만, 색욕) | 3 | 박탈감 × 집착의 충돌 |
| **분노** | 1 (시기) | 2 (교만, 나태) | 3 | 공격성 × 무기력의 충돌 |
| **나태** | 1 (폭식) | 2 (교만, 분노) | 3 | 수동적 향락 계열 |
| **탐욕** | 1 (색욕) | 2 (교만, 폭식) | 3 | 쌓음 × 소비의 충돌 |
| **폭식** | 1 (나태) | 2 (교만, 탐욕) | 3 | 수동적 향락 계열 |
| **색욕** | 1 (탐욕) | 2 (교만, 시기) | 3 | 욕망 × 박탈감의 충돌 |

### 서사 근거 (보존)

| 페어 | 관계 | 해석 |
|---|---|---|
| 분노 ↔ 시기 | 강화 | 박탈감 → 적개심 (카인/아벨 모티프) |
| 탐욕 ↔ 색욕 | 강화 | 물질·육체 욕망의 동맹 |
| 폭식 ↔ 나태 | 강화 | 수동적 향락 계열 동류 |
| 분노 ↔ 나태 | 대립 | 공격성 vs 무기력 |
| 탐욕 ↔ 폭식 | 대립 | 쌓음 vs 소비 |
| 시기 ↔ 색욕 | 대립 | 박탈감 vs 집착 대상 |
| 교만 vs 전 죄종 | 완전 대립 | **"모든 죄의 근원"** — 루시퍼 추락 모티프 |

### 매트릭스가 게임에 미치는 영향

1. **bonds 변동 배율** (§2-B) — 사건 발생 시 두 영웅의 메인 죄종을 매트릭스에 대입해 변동량 배율 결정
2. **사기 modifier 가속/감속** — 같은 파티/같은 시설 배치 시, 두 영웅의 메인 죄종이 **대립**이면 사기 modifier "껄끄러운 동거" 가속, **강화**면 "친구와 함께" 가속

> **방향성 없음**: 매트릭스는 대칭적으로 읽는다. "A=교만, B=분노"든 반대든 **교만↔분노 = 대립**으로 일괄.

## 4. 폭주/구원 — hero_design.md 참조

폭주(사기 폭주 임계) / 구원(사기 구원 임계) 시스템 본문은 [hero_design.md §2-2](hero_design.md#2-2-사기morale-시스템--rimworld-mood-방식)에 통합됨. 본 절은 **죄종 시스템 관점에서의 요약**만 둔다.

### 폭주 트리거

```
영웅 사기 ≤ [balance.csv:morale_break_threshold] → 매 턴 폭주 확률 체크
```

### 폭주 양상은 메인 죄종 기반

같은 폭주 발동이라도 영웅마다 다른 양상:

| 메인 죄종 | 폭주 효과 (CSV 키) |
|---|---|
| 분노 | 도발 — 랜덤 영웅 사기 -10 (`[balance.csv:rampage_wrath_morale_drain]`) |
| 교만 | 명령 거부 — 지시 무시 (`[balance.csv:rampage_pride_defiance_chance]`) |
| 탐욕 | 횡령 — 식량/골드 차감 (`[balance.csv:rampage_greed_steal_food]` / `_gold`) |
| 시기 | 사보타주 — 타 영웅 사기 -8 (`[balance.csv:rampage_envy_sabotage]`) |
| 색욕 | 집착 — 대상 분리 거부 (`[balance.csv:rampage_lust_obsession_refuse_chance]`) |
| 폭식 | 과식 — 추가 식량 소비 (`[balance.csv:rampage_gluttony_extra_food_cost]`) |
| 나태 | 태업 — 주변 영웅 사기 -3 전파 (`[balance.csv:rampage_sloth_propagation]`) |

> 시작특성 변형은 [hero_design.md §2-2](hero_design.md#2-2-사기morale-시스템--rimworld-mood-방식) 참조.

### 구원 트리거 — 사기 고양 영역

```
영웅 사기 ≥ [balance.csv:morale_inspiration_threshold] → 매 턴 구원 확률 체크
```

### 구원 보상 — 메인 죄종별 (보존)

7죄종 임계 이벤트의 구원 분기는 **사기 구원 영역에서의 확률 발동**으로 흡수.

| 메인 죄종 | 구원 발동 시 |
|---|---|
| 분노 → 평정 | 후천 특성 "평정" (분노 폭주 면역, 사기 변동 안정) |
| 교만 → 현자 | 후천 특성 "현자" (지휘 보너스) |
| 탐욕 → 청빈 | 후천 특성 "청빈" (자원 수급 안정) |
| 시기 → 대범 | 후천 특성 "대범" (관계 회복) |
| 색욕 → 순결 | 후천 특성 "순결" (집착 해제) |
| 폭식 → 단식자 | 후천 특성 "단식자" (식량 효율) |
| 나태 → 명상가 | 후천 특성 "명상가" (Stamina 회복) |

> 구원 특성 7종은 `traits.csv` (category=구원, type=salvation) 보존.

### 연쇄 반응 규칙

폭주 이벤트의 **결과**가 다른 영웅의 **트리거**가 되는 구조 (사기 modifier 부여 → 사기 임계 진입 → 추가 폭주 가능).

- **연쇄 깊이 제한**: 한 턴 내 최대 `[balance.csv:max_chain_reactions]`회
- **Catharsis 안전장치**: 폭주 직후 1턴 면역 (사기 +30 modifier 자동 부여)
- 폭주 이력 임계: `[balance.csv:rampage_history_threshold]`회 이상 폭주한 영웅은 후천 폭주 특성 부여 (Lv5/Lv10 분기)

## 5. 국시(國是, Edict) 시스템 — 보존 (2026-04-17 도입)

### 설계 철학 (변경 없음)

1. **단일 레버** — 바알은 누적 수치 없음. 7죄종 중 **1개 국시**만 선포하여 통치 색깔 표현
2. **기간형** — 선포 시 `[balance.csv:edict_duration]` 턴 지속, 이후 자동 만료
3. **공백 리듬** — 만료 후 `[balance.csv:edict_cooldown_normal]` 턴 공백
4. **자유 선포** — 해금 조건 없음
5. **대립 전환 비용** — 대립 죄종 국시 간 전환 시 추가 공백
6. **조기 해제 가능** — 만료 전 해제 시 페널티
7. **교만은 독립** — 대립 국시 없음

### 죄종 × 정책 영역 매핑 (보존)

| 죄종 | 국시명 | 정책 영역 | 대립 국시 |
|------|-------|----------|---------|
| 분노 | 강병령 | 군사 — 전쟁, 처벌 | 안식령 |
| 탐욕 | 축재령 | 경제 — 축적, 착취 | 풍요령 |
| 교만 | 위광령 | 위상 — 독선, 엘리트 | (없음) |
| 시기 | 경쟁령 | 경쟁 — 차등, 감시 | 열정령 |
| 나태 | 안식령 | 노동 — 방치, 안식 | 강병령 |
| 폭식 | 풍요령 | 복지 — 소비, 풍요 | 축재령 |
| 색욕 | 열정령 | 선동 — 관계, 열정 | 경쟁령 |

### 국시별 설계 의도 (사기 시스템 기준 재정의)

| 국시 | 죄종 | 효과 |
|------|---|---------|
| **강병령** | 분노 | 전투력 ↑ + 분노 메인 영웅 사기 가중 변동 |
| **축재령** | 탐욕 | 자원 수급 ↑ + 탐욕 메인 영웅 사기 가중 변동 |
| **풍요령** | 폭식 | 식량 소비 ↑ + 폭식 메인 영웅 사기 +modifier |
| **안식령** | 나태 | 생산 ↑ + 나태 메인 영웅 Stamina 회복 가속 |
| **경쟁령** | 시기 | 성장 가속 + 시기 메인 영웅 정찰 보너스 |
| **열정령** | 색욕 | bonds 변동 배율 ↑ |
| **위광령** | 교만 | 엘리트 강화 + 교만 메인 영웅 지휘 보너스 |

> **효과 수치 SSOT**: `src/data/edicts.csv` — 컬럼 갱신 필요 (이전 `morale_tick`/`morale_swing_mult` 컬럼은 사기 시스템 환원 전제로 그대로 활용 가능).

### 상태 전이 규칙 (보존)

```
무(無) 국시 → [선포] → 선포 중(edict_duration턴) → [자동 만료] → 쿨다운(edict_cooldown_normal턴) → 무(無) 국시
                            │
                            └─ [조기 해제] → 쿨다운(edict_cooldown_early턴) → 무(無) 국시

대립 국시 전환 시: 쿨다운 + edict_cooldown_opposite_bonus 턴 추가
```

### 데이터 구조 (보존)

```
Store.edict:
  null                                                    — 무 국시, 쿨다운 없음
  { type: 'wrath', activatedDay: 10, expireDay: 15,
    cooldownUntilDay: null }                              — 선포 중
  { type: null, activatedDay: null, expireDay: null,
    cooldownUntilDay: 17 }                                — 쿨다운 중
```
