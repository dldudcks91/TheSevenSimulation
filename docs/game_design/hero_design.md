# 영웅 시스템 설계

> 상태: 확정 (죄종 수치화 + 특성 시스템 + 주 성향 고정)
> 작성일: 2026-03-24 (최종 동기화: 2026-04-13)

---

## 1. 개요

영웅은 **인간**이다. 용병, 모험가, 떠돌이 — 바알에게 고용된 자들.
각자 깊은 성격적 결함을 안고 있으며, 그 결함이 **죄종 수치**로 표현된다.

멀쩡한 인간은 악마 밑에서 일하지 않는다.
갈 곳 없는 자, 버림받은 자, 망가진 자들이 바알에게 모인다.

### 죄종 수치 = 성격 차원

기존의 "죄종 태그 1개" 방식을 폐기하고, **7죄종을 수치 스탯**으로 전환한다.
모든 영웅은 7가지 죄종 수치를 가지며, **각 수치에 비례해 만족/불만 강도가 결정**된다. **최고 수치 = 주 성향**은 사기 양극단(0/100)에서의 행동 선택과 수식어 표기에 쓰인다.

| 죄종 수치 | 높을 때 (성격) | 낮을 때 (성격) |
|---------|--------------|--------------|
| **분노** (공격성) | 충동적, 폭력적, 참을성 없음 | 온순, 전투 소극적 |
| **시기** (비교심) | 비교 강박, 열등감, 뒷공작 | 타인에 무관심, 자족 |
| **탐욕** (소유욕) | 독점욕, 소유욕, 나누지 못함 | 무욕, 나눔 |
| **나태** (무기력) | 무기력, 회피, 전염되는 체념 | 근면, 자발적 |
| **폭식** (탐닉) | 과잉, 불안한 비축, 끝없는 허기 | 절제, 소식 |
| **색욕** (집착) | 집착, 의존, 관계 중독 | 냉정, 독립적 |
| **교만** (자존) | 자기확신, 독선, 복종 거부 | 겸손, 순종적 |

Phase 1에서는 7체 구현. 랜덤 생성.

## 2. 영웅 구조

```
[영웅] — 반고정 생성 (이름+고유 특성 고정, 죄종/스탯 랜덤)
├── 이름 = "수식어" + 이름 (수식어: 죄종 수치 상위 2개 조합, 이름: 21명 고정 풀)
├── 고유 특성 1개 (영웅 이름별 고정, hero_names.csv 매핑) — 장점 특화, 역할 정의
├── 죄종 7수치 (각 1~20, 랜덤) — 성격 + 만족/불만 + 폭주/이탈(최고값=주 성향) + 영웅 간 궁합 + 수식어
├── 사기 (0~100) — 유일한 관리 수치
├── 기본 스탯 7개 (각 1~20, 랜덤) — 전투/작업 능력 결정
├── 파생 세부스탯 — 기본 스탯에서 산출 (행동 + 관계)
├── 후천 특성 최대 2개 (Lv5, Lv10 레벨업 시 행동 XP or 폭주 이력 기반)
├── 레벨 (1~10) + XP (카테고리별 누적)
├── acquiredTraits[] — 후천 특성 배열
├── rampageHistory{} — 폭주 유형별 횟수 기록
├── 히든: 성장률 + 상한선 (스탯별)
└── 장비 3슬롯 — Phase 2
```

> **같은 영웅은 1회 플레이에 중복 등장하지 않음.** 21명 풀에서 7명을 뽑으므로 3회차면 전원 조우 → 메타 지식 형성.

> **UI**: 영웅 고용 시 기본 스탯 + 주 성향(최고 죄종) + 특성 표시. 영입 후 상세 확인 가능.

## 2-1. 스탯 시스템

### 설계 원칙
- **죄종 수치 = 성격**, **기본 스탯 = 능력** — 완전 분리
- 죄종 수치와 기본 스탯 모두 랜덤 생성, 서로 독립
- 한 스탯이 여러 행동에 영향 → 배치 딜레마 발생
- UI에서 행동별 적합도를 별점으로 자동 표시

### 기본 스탯 7개 (각 1~20)

| 스탯 | 의미 | 영향 |
|------|------|------|
| **힘** | 물리적 파워 | 사냥, 건설, 대장간, 전투 |
| **민첩** | 빠르기, 손재주 | 사냥, 채집, 연금술, 전투 |
| **지능** | 머리, 판단력 | 외교, 연구(단독), 교역, 전투 |
| **건강** | 튼튼함, 면역력 | 체력 최대치/회복 결정 + 발병 저항 |
| **감각** | 눈썰미, 직감 | 채집, 대장간, 연금술 |
| **통솔** | 지휘, 리더십 | 건설, 통솔력(관계), 전투 |
| **매력** | 영향력, 교섭 | 외교, 교역, 고용(비용 할인), 매력도(관계) |

### 체력 시스템

영웅은 사기 외에 **체력**(1~100) 게이지를 가진다. UI에서 사기 바 위에 표시.

| 항목 | 값 |
|------|-----|
| **범위** | 1~100 |
| **초기값** | 100 |
| **회복** | 건강 스탯에 비례 (balance.csv) |
| **소모** | 행동/전투/부상 시 감소 |

#### 체력 구간별 상태

| 구간 | 상태 | 효과 |
|------|------|------|
| 51~100 | 정상 | - |
| 26~50 | 피로 | 행동 효율 -20% |
| 1~25 | 과로 | 매 턴 **발병 확률** 체크 (건강 높으면 확률↓) |
| 0 | 탈진 | 행동 불가, 강제 휴식 |

#### 발병

- 과로 상태에서 매 턴 확률 판정 → 발병 시 **2~3턴 행동 불가** + 사기 하락
- 병원 있으면 1턴으로 단축
- 건강 스탯이 높을수록 발병 확률↓

> 건강 스탯의 의미 = **"체력 최대치/회복 + 발병 저항"**

### 체력 시스템 구현 현황 (2026-04-16)

| 항목 | 상태 | 비고 |
|------|------|------|
| hero.stamina 필드 (1~100, 기본 100) | ✅ | HeroManager._generateHero |
| 턴당 회복 `base + health×bonus` | ✅ | TurnProcessor.processDayPhase |
| 행동 소모 (채집/벌목/사냥/원정) | ✅ | balance.csv `stamina_cost_*` |
| 피로/과로 구간 효율 감소 (채집/벌목 ×0.8/×0.6) | ✅ | DayActions._efficiencyMult |
| 과로 시 매턴 발병 확률 판정 | ✅ | HeroManager.checkSicknessTick |
| 발병 → `status='sick'`, 2~3턴 행동 불가 | ✅ | sickTurns 카운터 |
| UI — MapHeroInspector 체력 바 + 라벨 | ✅ | 정상/피로/과로/탈진 |
| 병원 있을 시 발병 기간 단축 | ⬜ | 미구현 (추후) |

### 전투 내구력

- 영웅 HP = `hero_hp_base(50)` + 건강(vitality) × `hero_hp_per_vitality(5)` (기본 vitality 10 → HP 100)
- 영웅이 직접 피격. HP 0 → `knocked_out` → 귀환 시 `injured`
- 매 턴 HP 자연 회복 `hero_hp_regen_per_turn(10)`. **병원** 회복 턴 절반, **훈련장** 회복 +50%, **치료약** 즉시 회복

### 죄종 × Stamina 보정 (2026-04-16 추가)

주 성향(topSin)에 따라 거점 행동의 Stamina 소모/회복이 보정된다. 누적 없이 1개만 적용.

| 주 성향 | 소모 배율 | 회복 배율 | 해석 |
|---------|----------|----------|------|
| 나태(sloth) | 0.8 | 1.2 | 쉬엄쉬엄 일하지만 회복은 빠르다 |
| 분노(wrath) | 1.1 | 1.0 | 몰아쳐 일하므로 피로 빨리 누적 |
| 시기(envy) | 1.1 | 1.0 | 과업 집착으로 피로 누적 |
| 폭식(gluttony) | 1.0 | 1.1 | 잘 먹어서 회복 +10% |
| 교만(pride) | 1.0 | 0.9 | 자존심이 휴식을 막음 |
| 탐욕(greed) / 색욕(lust) | 1.0 | 1.0 | 기본 |

- 실제 키: `balance.csv`의 `stamina_mult_<sin>_cost` / `stamina_mult_<sin>_recover`
- 적용 지점: `HeroManager.consumeStamina` / `HeroManager.recoverStaminaTurn` (주 성향 1개 기준)

### Stamina → 사기 페널티 (2026-04-16 추가)

| 상태 | 조건 | 사기 변화 | balance.csv 키 |
|------|------|----------|---------------|
| 탈진 | Stamina = 0 | -10 (즉시) | `morale_penalty_exhausted` |
| 과로 | Stamina 1~25 지속 | -2 /턴 | `morale_penalty_overwork_tick` |
| 발병 | 과로 상태에서 발병 트리거 | -8 | `sickness_morale_delta` |

적용 지점: `TurnProcessor.processDayPhase` — 매 턴 Stamina 구간 체크 후 `updateMorale`.

모든 행동은 **동등 가중치(50:50)**. 행동 효율 = (스탯A + 스탯B) / 2. 연구만 지능 단독.

| **힘** | ? | ● | | | ● | ● | | | | |
| **민첩** | ? | ● | ● | | | | ● | | | |
| **지능** | ? | | | ● | | | | ● | ● | |
| **건강** | - | - | - | - | - | - | - | - | - | - |
| **감각** | ? | | ● | | | ● | ● | | | |
| **통솔** | ? | | | | ● | | | | | |

전투: 힘+민첩(원정), 힘+통솔(방어). 건강은 행동 매핑 없음 — 체력 게이지(최대치/회복) 전용.

### 파생 세부스탯 — 관계 계열

| 세부스탯 | 산출 | 관계에서의 역할 |
|---------|------|---------------|
| **통솔력** | 통솔 | 다른 영웅이 이 영웅을 **따름** (존경) |
| **매력도** | 매력 | 다른 영웅이 이 영웅을 **좋아함** (호감) |
| **민감도** | 색욕 수치 (죄종) | 이 영웅이 다른 영웅에게 **민감하게 반응** |

> 민감도는 기본 스탯 감각이 아닌, 죄종 수치 중 색욕(lust)에서 산출.

## 2-2. 죄종 수치 (성격 시스템)

### 설계 원칙
- 기존 "독립 세부스탯 7개"를 **죄종 7수치**로 교체
- 기본 스탯과 무관, 랜덤 생성 (각 1~20)
- **높다고 무조건 좋지 않음** — 양면성을 가짐
- 만족/불만 강도는 각 죄종 수치에 비례 (모든 수치가 동시에 영향) — **이것이 죄종 시스템의 일상 효과**
- **최고 수치 = 주 성향** — 사기 양극단(0/100)에서의 행동 선택과 수식어 표기에 사용
- 죄종 수치 간 관계 매트릭스(생성/강화/대립)가 영웅 내부+영웅 간 모두 적용

### 7개 죄종 수치

| 죄종 수치 | 높으면 | 낮으면 |
|---------|-------|-------|
| **분노** (공격성) | 전투 공격력↑, 동료 충돌 잦음 | 전투 소극적, 거점 내 평화 |
| **탐욕** (소유욕) | 보상 획득량↑, 자원 갈등 | 수확 적지만 나눔 |
| **교만** (자존) | 리더 시 능력↑, 부하 역할 시 사기↓ | 어디든 불만 없이 배치 |
| **시기** (비교심) | 탐색/연구 보너스, 위험 이벤트 조우↑ | 안전하지만 발견 기회 적음 |
| **폭식** (탐닉) | 장기 작업 보너스, 중단/철수 판단 느림 | 판단 빠르지만 오래 못 버팀 |
| **색욕** (집착) | 관계 형성 빠름, 동료 사기에 같이 흔들림 | 냉정, 연쇄 전염 저항 |
| **나태** (무기력) | 혼자 행동 효율↑, 팀 시너지↓ | 팀에서 잘 맞지만 혼자선 약함 |

### 폭주/이탈 행동 결정 방식 (주 성향 고정 + 특성 예외)

1. **행동 강박 특성 우선** — "방화광", "파괴충동", "도벽" 등 행동 강박 특성이 있으면 해당 강박 행동이 **우선 발동**
2. **주 성향 고정** — 특성 예외가 없으면, **최고 죄종 수치(주 성향)**에 따라 행동이 **고정 결정**

> 수식어 결정에는 상위 2개 죄종 조합을 사용 (이름 생성용)
> 예: 분노+탐욕 높음 = "약탈자" 수식어 (전투 + 자원 집착 성격)

## 2-3. 특성 시스템

### 개요
영웅은 **고유 특성 1개(이름별 고정)**와 **후천 특성 최대 2개(Lv5, Lv10)**를 가진다.

```
고유 특성 1개 (영웅 이름별 고정, hero_names.csv 매핑) — 장점 전용
+ 후천 특성 1개 (Lv5 도달 시, 행동 XP 기반 or 폭주 이력 기반)
+ 후천 특성 1개 (Lv10 도달 시, 행동 XP 기반 or 폭주 이력 기반)
= 최대 3개
```

### 설계 원칙: 역할 분리 (DD 모델)

| 장치 | 담당 | 예시 |
|------|------|------|
| **고유 특성** | 장점 (역할 정의) | "카엘 = 전투 데미지 +15%" |
| **죄종 수치** | 단점 (드라마 생성) | "분노 주 성향 → 미파견 시 불만" |
| 조합 | 매 플레이 다른 경험 | 같은 카엘이라도 죄종에 따라 다른 약점 |

> 고유 특성에 단점을 넣지 않는 이유: 죄종 시스템이 이미 DD의 "-퀴크" 역할을 충분히 함. 특성까지 단점을 넣으면 이중 처벌 + 기획/구현 비용 2~3배.
> 행동강박 특성의 "폭주 시 특수 행동"은 평소 단점이 아니라 극단 상황 변수이므로 유지.

### 고유 특성 카테고리

| 카테고리 | 개수 | 설명 |
|---------|------|------|
| 일반 장점 | 14개 | 특정 행동/상황에서 보너스 (상황 의존적, 항상 최선 아님) |
| 행동 강박 | 7개 | 평소 장점 + 폭주/이탈 시 특수 행동 |
| 후천 행동 | 10개 | 레벨업 시 행동 XP 기반 획득 |
| 후천 폭주 | 7개 | 레벨업 시 폭주 이력 기반 획득 |

### 고유 특성 21종 (영웅별 고정, 장점 전용)

#### ⚔️ 전사 계열 (7종) — 전투/원정/사냥

| 영웅 | 특성 | 효과 |
|------|------|------|
| 카엘 | 호전적 | 전투 데미지 +15% |
| 모르 | 강인한 | 전투 피해 -20% |

### 행동강박 특성 (7종, 고유 특성과 별도)

행동강박 특성은 위 21명에 포함되지 않는 **별도 시스템**이 아니라, 아래 표의 영웅들이 가진 고유 특성이다. 평소에는 장점을 제공하지만, **폭주/이탈 시 특수 행동**이 발동된다.

> 행동강박은 "평소 단점"이 아니라 "극단 상황 변수"이므로 장점 전용 원칙에 부합.

위 21명 중 행동강박 보유 영웅은 없다. 행동강박은 **후천 폭주 특성(Lv5/Lv10)**으로만 획득 가능하도록 변경.

> 설계 변경: 기존 "선천 행동강박" 폐기 → 행동강박은 후천 폭주 특성에 통합. 폭주를 반복한 영웅만 강박 행동이 고정되는 구조.

### 후천 특성 (Lv5, Lv10 레벨업 시 획득)

레벨업 시점에 **행동 XP 1위 카테고리**에 해당하는 특성을 자동 획득.
단, **폭주 이력이 2회+**인 유형이 있으면 폭주 특성이 우선 부여된다.

```
Lv5 도달 → 폭주 이력 2회+ 있으면 → 해당 폭주 특성
             폭주 이력 없으면    → 행동 XP 1위 카테고리 특성

Lv10 도달 → 동일 로직 (남은 슬롯에 2위 카테고리 or 2번째 폭주)
```

> 부수 효과: 폭주가 반복되면 후천 슬롯 일부가 폭주 특성으로 채워져, 행동 XP 보상이 그만큼 줄어든다.

## 2-4. 성장 시스템 (레벨 + XP)

### 개요
행동을 할 때마다 **경험치(XP)**를 얻고, 총 XP가 쌓이면 **레벨업**한다.
레벨업 시 **가장 많이 쌓인 XP 카테고리**의 관련 스탯이 상승한다.
최대 레벨은 **10**.

### XP 카테고리

| XP 카테고리 | 획득 행동 | 레벨업 시 상승 스탯 |
|-----------|----------|-----------------|
| 전투 XP | 원정, 방어전 전투 | 힘, 민첩 |
| 방어 XP | 방어전 배치 | 건강, 통솔 |
| 노동 XP | 채집, 벌목 | 민첩, 감각 |
| 건설 XP | 건설 | 힘, 통솔 |
| 연구 XP | 연구 | 지능 |
| 사냥 XP | 사냥 | 힘, 민첩 |

### 레벨업 흐름

```
행동 → 해당 카테고리 XP 획득 → 총 XP 누적
  → 레벨업 임계값 도달 → 레벨 상승
    → XP 1위 카테고리의 관련 스탯 +1~2
    → Lv5: 후천 특성 1개 획득
    → Lv10: 후천 특성 1개 획득 (총 2개)
```

### 레벨업 보상

| 레벨 | 보상 |
|------|------|
| Lv2~4, 6~9 | XP 1위 카테고리 관련 스탯 +1 |
| **Lv5** | 스탯 +1 + **후천 특성 1개** (폭주 이력 우선 → 행동 XP 기반) |
| **Lv10** | 스탯 +1 + **후천 특성 1개** (동일 로직, 2위 카테고리) |

### 후천 특성 획득 판정 (Lv5, Lv10)

```
1. 폭주 이력 2회+인 유형이 있는가?
   → YES: 해당 폭주 특성 부여 (행동 특성 대신)
   → NO: 행동 XP 1위 카테고리의 후천 특성 부여
2. Lv10에서는 남은 슬롯에 같은 로직 반복 (2위 카테고리 or 2번째 폭주)
```

> 부수 효과: 폭주가 반복되면 후천 슬롯 일부가 폭주 특성으로 채워져, 행동 XP 보상이 그만큼 줄어든다.

### 히든 파라미터 (플레이어에게 비공개)

| 파라미터 | 의미 | 비고 |
|---------|------|------|
| **성장률** (스탯별) | 해당 스탯 상승 속도 | 같은 행동을 해도 영웅마다 성장 속도 다름 |
| **상한선** (스탯별) | 해당 스탯 최대 도달치 | 재능의 한계 — 상한이 20 이상이면 초월 성장 가능 |

### 딜레마 예시

```
원정만 보내던 영웅이 Lv5 도달
  → 전투 XP 1위 → "역전의 용사" 획득!
  → 하지만 폭주 2회 경험했다면 → "파괴자" 획득... 전투 보상 날림

건설+연구를 골고루 시킨 영웅이 Lv5
  → 건설 XP가 근소하게 1위 → "건축가" 획득
  → Lv10에서는 연구 XP가 2위 → "연구자" 획득 → 만능형 성장

채집만 시킨 영웅이 Lv5, Lv10 모두 노동 XP 1위
  → Lv5 "노동의 기쁨" → Lv10은 2위 카테고리 특성
```

## 3. Phase 1 영웅 (7체)

영웅은 **반고정 생성**. 이름+고유 특성은 고정, 죄종/스탯은 랜덤. 직업 없음 — 스탯과 장비가 역할을 정의.
- 이름: 21명 고정 풀에서 선택 (중복 불가) + 죄종 상위 2개 조합 수식어
- 고유 특성: 영웅 이름별 1:1 고정 (hero_names.csv 매핑, 장점 전용)
- 죄종 수치: 7개 각 1~20 랜덤
- 기본 스탯: 7개 각 1~20 랜덤
- 레벨: 1 (최대 10), XP: 카테고리별 0
- 후천 특성: Lv5, Lv10에서 각 1개 획득 (행동 XP or 폭주 이력 기반)

## 4. 죄종 수치별 만족/불만 조건

**주 성향(최고 죄종 수치)**이 만족/불만의 기본 틀을 결정하고, 부 성향도 수치에 비례하여 약한 강도로 영향한다.

| 죄종 | 만족 조건 (사기 상승) | 불만 트리거 (사기 하락) |
|---------|-------------------|---------------------|
| 교만 | 보스 원정 성공 | 3연속 원정 실패 |
| 탐욕 | 자원 투자받음 | 자원 배분 안 됨 |
| 분노 | 원정 파견 | 3일 이상 미파견 |
| 시기 | 본인 레벨업 | 타 영웅 더 성장 |
| 색욕 | 선호 동료 함께 배치 | 선호 동료 장기 분리 |
| 나태 | 시설 미배치 (휴식) | 연속 배치 |
| 폭식 | 식량 충분 | 식량 부족 |

> 예: 분노(주 성향) 15 + 탐욕 8인 영웅은 원정 파견 시 분노 기반 강한 만족 + 자원 배분 시 탐욕 기반 약한 만족. 주 성향이 지배적이고 부 성향도 보조적으로 작용.

- 수식어 = **죄종 수치 상위 2개의 교차점**
- "약탈자" = 분노 + 탐욕 높음 → 전투 강하고 보상 많지만, 충돌 + 자원 갈등
- "고독한" = 색욕 + 나태 높음 → 혼자 강하지만 관계에 흔들림
- 같은 주 성향이라도 부 성향에 따라 **완전히 다른 수식어** → 개성 부여
