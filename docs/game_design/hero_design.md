# 영웅 시스템 설계

> 상태: **메인 죄종 고정 + 사기(Morale) 1~100 시스템** 으로 전면 개편 (2026-04-27)
> 작성일: 2026-03-24 | 마지막 업데이트: 2026-04-27 (Phase 메인-사기 롤백)

---

## 1. 개요

영웅은 **인간**이다. 용병, 모험가, 떠돌이 — 바알에게 고용된 자들.
멀쩡한 인간은 악마 밑에서 일하지 않는다. 갈 곳 없는 자, 버림받은 자, 망가진 자들이 바알에게 모인다.

각 영웅은 **결함 있는 인간**으로서 7대 죄악 중 하나의 메인 죄종을 짊어진다. 그 죄종이 폭주의 양상을 결정한다.

### 정체성 모델 (2026-04-27 확정)

생성 시 영웅이 가지는 3가지 정체성 요소:

```
[영웅 = 1이름 + 1메인 죄종(고정) + 1시작특성(고정)]
```

- **이름**: 21명 고정 풀에서 선택 (중복 불가). 같은 영웅은 1회 플레이에 중복 등장하지 않음
- **메인 죄종**: 영웅 이름별 1:1 고정 (`hero_names.csv`에 매핑). 폭주 양상을 결정하는 정체성 표지
- **시작특성**: 영웅 이름별 1:1 고정 (`traits.csv`). 능력/사기/폭주 양상을 다축으로 변형

> 운영 중 변하는 건 **사기·체력·HP·후천특성·관계·XP**. 위 3가지 정체성 요소는 영구 불변.

### 폐기된 개념 (2026-04-27)

- 7죄종 누적 게이지 (sinStats)
- topSin (현재 최고 죄종) 동적 정체성
- sinPriority (숨은 우선순위 순열)
- 4구간 시스템 (깨끗/발현/고양/폭주/임계)
- 죄종별 정화 수단 카탈로그
- 수식어 시스템 (`hero_epithets.csv`)
- 시작특성 편향/저항/중립 분류

## 2. 영웅 구조

```
[영웅] — 반고정 생성 (이름+메인죄종+시작특성 고정, 스탯 랜덤)
├── 이름 (hero_names.csv 21명 풀)
├── 메인 죄종 1개 (이름별 고정, hero_names.csv main_sin 컬럼)
├── 시작특성 1개 (이름별 고정, hero_names.csv 매핑) — traits.csv 참조
├── 기본 스탯 7개 (각 [balance.csv:stat_min] ~ [balance.csv:stat_max], 랜덤)
├── 사기 (Morale) — 1~100 단일 게이지, 기본값 50
│   └── modifier 합산식 (memory + situational, 사기 modifier 카탈로그 참조)
├── Stamina — 체력 게이지
├── HP — 전투 내구력
├── 후천특성 최대 2개 (Lv5, Lv10 레벨업 시)
├── 레벨 (1 ~ 10) + XP (카테고리별 누적)
├── acquiredTraits[] — 후천 특성 배열
├── rampageHistory{} — 폭주 유형별 횟수 기록
├── bonds{} — 다른 영웅에 대한 호감도 (asymmetric)
├── 히든: 성장률 + 상한선 (스탯별)
└── 장비 3슬롯 — Phase 2
```

> **특성 3슬롯**: 시작특성 1 + 후천특성 2 = 최대 3개. 시작특성이 "정체성", 후천특성이 "플레이어 선택의 성장".

> **UI**: 영웅 고용 시 이름·메인 죄종·시작특성·기본 스탯 표시. 사기는 50에서 시작, 행동/사건/관계에 따라 변동.

## 2-1. 스탯 시스템

### 설계 원칙
- **죄종 = 정체성 (성격/폭주 양상)**, **기본 스탯 = 능력**, **사기 = 상태** — 완전 분리
- 한 스탯이 여러 행동에 영향 → 배치 딜레마 발생
- UI에서 행동별 적합도를 별점으로 자동 표시

### 기본 스탯 7개 (각 `[balance.csv:stat_min]` ~ `[balance.csv:stat_max]`)

| 스탯 | 의미 | 영향 |
|------|------|------|
| **힘** | 물리적 파워 | 사냥, 건설, 대장간, 전투 |
| **민첩** | 빠르기, 손재주 | 사냥, 채집, 연금술, 전투 |
| **지능** | 머리, 판단력 | 외교, 연구(단독), 교역, 전투 |
| **건강** | 튼튼함, 면역력 | 체력 최대치/회복 결정 + 발병 저항 |
| **감각** | 눈썰미, 직감 | 채집, 대장간, 연금술 |
| **통솔** | 지휘, 리더십 | 건설, 전투 |
| **매력** | 영향력, 교섭 | 외교, 교역, 고용(비용 할인) |

### 행동별 스탯 매핑 (보존)

모든 행동은 **동등 가중치(50:50)**. 행동 효율 = (스탯A + 스탯B) / 2. 연구만 지능 단독.

|  | 야외 |||| 실내 ||||||
|---|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
|  | 전투 | 사냥 | 채집 | 외교 | 건설 | 대장간 | 연금술 | 연구 | 교역 | 고용 |
| **힘** | ? | ● | | | ● | ● | | | | |
| **민첩** | ? | ● | ● | | | | ● | | | |
| **지능** | ? | | | ● | | | | ● | ● | |
| **건강** | - | - | - | - | - | - | - | - | - | - |
| **감각** | ? | | ● | | | ● | ● | | | |
| **통솔** | ? | | | | ● | | | | | |
| **매력** | ? | | | ● | | | | | ● | ● |

전투: 힘+민첩(원정), 힘+통솔(방어). 건강은 행동 매핑 없음 — 체력 게이지(최대치/회복) 전용.

### 체력(Stamina) 시스템 — 보존

영웅은 **체력(Stamina)** 게이지를 가진다. UI에서 영웅 카드에 표시.

| 항목 | 값 (CSV 키) |
|------|-----|
| **최대치** | `[balance.csv:stamina_max]` |
| **초기값** | 최대치 |
| **턴당 회복** | `[balance.csv:stamina_recovery_base] + 건강 × [balance.csv:stamina_recovery_health_bonus]` |
| **소모** | 행동별 키 (`stamina_cost_gather` / `_lumber` / `_hunt` / `_build` / `_research` / `_feast` / `_expedition`) |

#### 체력 구간별 상태

| 구간 (경계 키) | 상태 | 효과 |
|------|------|------|
| `> stamina_tired_threshold` | 정상 | - |
| `stamina_overwork_threshold` ~ `stamina_tired_threshold` | 피로 | 행동 효율 감소 |
| `0` ~ `stamina_overwork_threshold` | 과로 | 매 턴 발병 확률 체크 |
| `0` | 탈진 | 행동 불가 |

> 건강 스탯의 의미 = **"체력 최대치/회복 + 발병 저항"**

### 전투 내구력

- HP 공식: `[balance.csv:hero_hp_base] + 건강 × [balance.csv:hero_hp_per_vitality]`
- 영웅이 직접 피격. HP 0 → `knocked_out` → 귀환 시 `injured`
- 매 턴 HP 자연 회복: `[balance.csv:hero_hp_regen_per_turn]`. 병원/훈련장/치료약은 시설별 배율 (facilities.csv)

## 2-2. 사기(Morale) 시스템 — RimWorld Mood 방식

### 개요

영웅의 정신 상태를 표현하는 **단일 게이지**. 1~100 범위, 기본값 50. 폭주/구원/이탈의 트리거.

```
사기(Morale) = 기본값(50) + Σ(활성 modifier.offset)
범위: clamp(0, 100)
```

### 핵심 설계: modifier 합산식

사기는 단일 숫자지만 내부적으로는 **modifier(=Thought) 객체들의 합산**. UI에 분해해서 표시.

```
modifier {
  source:   'combat_victory' | 'judge_unfavored' | 'starter_trait' | ...
  category: 'memory' | 'situational' | 'permanent'
  offset:   ±정수 (사기 변동량)
  ttl:      만료까지 남은 턴 (memory) / 조건 ID (situational) / null (permanent)
  stack:    현재 누적 횟수 (stackLimit, stackedEffectMultiplier 적용)
}
```

| 카테고리 | 발생 | 소멸 | 예시 |
|---|---|---|---|
| **memory** | 사건 발생 시 추가 | TTL 만료 시 자동 제거 | "전투 승리" +5 (3턴) |
| **situational** | 조건 진입 시 활성 | 조건 이탈 시 비활성 | "전장 배치" -3 (배치 해제까지) |
| **permanent** | 정체성 부여 시 | 영웅 소멸까지 | "낙천적 시작특성" +12 |

> modifier 카탈로그(어떤 사건이 어떤 offset을 부여하는가)는 §4 참조.

### 임계값 — 폭주와 구원

```
사기 100 ─┬─ [구원 영역 80~99] — 매 턴 구원 확률 체크 (RimWorld Inspiration 방식)
        50 ─┼─ 정상 영역 (변동만)
        0 ─┴─ [폭주 영역 0~30] — 매 턴 폭주 확률 체크 (RimWorld Mental Break 방식)
```

| 영역 | 사기 범위 (CSV 키) | 효과 |
|---|---|---|
| **구원** | `≥ [balance.csv:morale_inspiration_threshold]` | 매 턴 확률 체크. 발동 시 **구원 이벤트** (메인 죄종별 분기) |
| **고양** | `[balance.csv:morale_high_threshold]` ~ `morale_inspiration_threshold` | 능력 보너스 (스탯 +1, 행동 효율 ↑) |
| **정상** | `morale_low_threshold` ~ `morale_high_threshold` | 변동만, 효과 없음 |
| **불안** | `morale_break_threshold` ~ `morale_low_threshold` | 능력 페널티 (행동 효율 ↓) |
| **폭주** | `≤ [balance.csv:morale_break_threshold]` | 매 턴 확률 체크. 발동 시 **폭주 이벤트** (메인 죄종별 분기) |

> 임계 4개와 영역 5개의 경계는 모두 `balance.csv` 키. 본문은 의미 표현만.

### 폭주/구원 확률 곡선 (RimWorld 참고)

선형이 아닌 **곡선** — 임계에서 멀수록 확률 급증.

```
사기 30 (폭주 임계점):  매 턴 ~5%
사기 20:               매 턴 ~15%
사기 10:               매 턴 ~30%
사기 0:                매 턴 ~50%+
```

> 실제 곡선은 `[balance.csv:morale_break_chance_curve_*]` 키로 관리.

### Catharsis (안전장치)

폭주 종료 시 **카타르시스 modifier(+30 사기, 1턴)** 자동 부여. 폭주 직후 즉시 재폭주를 방지.

### 폭주 양상 — 메인 죄종 × 시작특성 변주

같은 폭주 상태(사기 임계 도달)이어도 **메인 죄종**이 양상의 큰 틀을 결정하고, **시작특성**이 변형한다.

| 메인 죄종 | 기본 폭주 양상 | 시작특성 변형 예 |
|---|---|---|
| 분노 | Berserk — 아군 공격 / 적 추격 강박 | 호전적: 적 추격 강박 / 잔인한: 시체 훼손 |
| 교만 | 명령 거부 / 단독 행동 | 야행성: 밤에만 발현 / 카리스마: 다른 영웅 선동 |
| 탐욕 | 자원 횡령 / 창고 약탈 | 수집광: 창고 약탈 / 행운아: 도박 강박 |
| 시기 | 사보타주 / 타 영웅 약화 시도 | 정찰병: 적과 내통 시도 |
| 색욕 | 배치 거부 / 특정 영웅 집착 | 사교적: 특정 영웅 집착 |
| 폭식 | 식량 소진 / 폭식 폭주 | 사교적: 연회 강박 |
| 나태 | 행동 거부 / 무기력 전파 | 독불장군: 단독 출분 / 둔감한: 임계 +3 |

→ **메인 7 × 시작특성 21 = 147가지 폭주 양상**. 단조로움 회피.

### 구원 양상 — 메인 죄종별 보상

사기 고양 시 매 턴 구원 확률 체크. 발동 시 **메인 죄종별 구원 보상** 부여 (해당 죄종 저항 특성 등).

| 메인 죄종 | 구원 발동 시 효과 |
|---|---|
| 분노 → 평정 | 후천 특성 "평정" 획득 가능 (분노 폭주 면역) |
| 교만 → 현자 | 후천 특성 "현자" 획득 가능 (지휘 보너스) |
| 탐욕 → 청빈 | 후천 특성 "청빈" 획득 가능 (자원 수급 안정) |
| 시기 → 대범 | 후천 특성 "대범" 획득 가능 (관계 회복) |
| 색욕 → 순결 | 후천 특성 "순결" 획득 가능 (집착 해제) |
| 폭식 → 단식자 | 후천 특성 "단식자" 획득 가능 (식량 효율) |
| 나태 → 명상가 | 후천 특성 "명상가" 획득 가능 (Stamina 회복) |

> 구원 특성 7종은 `traits.csv` (category=구원) 보존.

## 2-3. 사기 ↔ 정체성 분리

| 차원 | 담당 | 변동성 |
|---|---|---|
| **메인 죄종** | 폭주 양상 결정 (정체성) | **불변** |
| **시작특성** | 능력/사기/폭주 양상 변형 | **불변** |
| **사기** | 현재 상태 (mood) | **동적 변동** |
| **사기 modifier** | 변동의 원인 (Thought) | 추가/만료 |

→ "이 카엘은 분노 영웅이다" (불변 정체성) + "이 카엘의 사기가 지금 32다" (동적 상태) → 직교.

## 2-4. 특성 시스템 (3슬롯)

### 개요
영웅은 **최대 3개 특성**을 가진다 — 시작특성 1 + 후천특성 2.

```
시작특성 1개 (영웅 이름별 고정, hero_names.csv 매핑)
+ 후천특성 1개 (Lv5 도달 시, 행동 XP or 폭주 이력 기반)
+ 후천특성 1개 (Lv10 도달 시, 행동 XP or 폭주 이력 기반)
= 최대 3개
```

### 시작특성 21종 (영웅별 1:1 고정 — 보존)

> **데이터 SSOT**: `src/data/traits.csv` (id, name, category, type, pro_effect, con_effect, target_sin, sin_mult).
> 메인 죄종 고정 전환에 따라 `target_sin`은 "이 특성이 영향을 주는 죄종"으로 의미 재정의 (편향/저항 분류 폐기).

| 모르 | 분노 | 강인한 | 전투 피해 -20% |

### 시작특성 = RimWorld Trait처럼 다축 영향

> **설계 원칙**: 시작특성은 단일 효과가 아니라 **사기/스킬/폭주 양상/사회 다축에 동시 영향**을 줄 수 있다. RimWorld의 Pyromaniac/Bloodlust/Sanguine 모델 차용.

예시 (확장 후 의도):
- **호전적**(카엘): 전투 데미지 +15% (스킬) + 분노 폭주 양상 변형(적 추격 강박)
- **낙천적**(유나): 영구 사기 +12 (modifier permanent) — RimWorld Sanguine 직역
- **예민한**(라엘): 사기 modifier offset 배율 +30% (양/음 모두 증폭)

> 다축 효과의 정확한 컬럼 구조는 `traits.csv` 확장 시 결정.

### 후천 특성 (Lv5, Lv10 레벨업 시 획득) — 보존

레벨업 시점에 **행동 XP 1위 카테고리**에 해당하는 특성을 자동 획득.
단, **폭주 이력이 2회+** 인 유형이 있으면 폭주 특성이 우선 부여된다.

```
Lv5 도달 → 폭주 이력 2회+ 있으면 → 해당 폭주 특성
             폭주 이력 없으면    → 행동 XP 1위 카테고리 특성

Lv10 도달 → 동일 로직 (남은 슬롯에 2위 카테고리 or 2번째 폭주)
```

#### 후천 행동 특성 (10종, 행동 XP 기반)

| 특성 | XP 카테고리 | 효과 |
|------|-----------|------|
| 역전의 용사 | 전투 XP 1위 | 원정 전투 데미지 +10%, 사기 변동 안정 |
| 수호자 | 방어 XP 1위 | 방어전 스탯↑ |
| 노동의 기쁨 | 노동 XP 1위 | 자원 작업 효율↑, 작업 시 사기 +1 |
| 연구자 | 연구 XP 1위 | 연구 속도↑ |
| 건축가 | 건설 XP 1위 | 건설 속도↑, 건설 비용↓ |
| 사냥꾼 | 사냥 XP 1위 | 사냥 성공률↑, 전리품↑ |
| 지옥 생존자 | (폭주 2회 경험) | 사기 변동폭↑↑, 전투력↑ |
| 개심자 | (구원 달성) | 사기 회복 +50% |

#### 후천 폭주 특성 (7종, 폭주 이력 2회+ 시 우선 부여)

같은 메인 죄종 폭주가 2회 반복되면 다음 레벨업에서 부여. 이후 해당 폭주가 **고정 발동**.

| 파괴자 | 분노형 폭주 2회 | 폭주 시 항상 건물 파괴, 전투 공격력 보정↑ |
| 약탈자 | 탐욕형 폭주 2회 | 폭주 시 항상 자원 도난, 보상 획득↑ |
| 반역자 | 교만형 폭주 2회 | 폭주 시 항상 명령 거부, 단독 행동 효율↑ |

| 탐식가 | 폭식형 폭주 2회 | 폭주 시 항상 식량 소진, 체력 회복↑ |

## 2-5. 성장 시스템 (레벨 + XP) — 보존

### 개요
행동을 할 때마다 **경험치(XP)**를 얻고, 총 XP가 쌓이면 **레벨업**한다.
레벨업 시 **가장 많이 쌓인 XP 카테고리**의 관련 스탯이 상승한다. 최대 레벨은 **10**.

### XP 카테고리

| XP 카테고리 | 획득 행동 | 레벨업 시 상승 스탯 |
|-----------|----------|-----------------|
| 전투 XP | 원정, 방어전 전투 | 힘, 민첩 |
| 방어 XP | 방어전 배치 | 건강, 통솔 |
| 노동 XP | 채집, 벌목 | 민첩, 감각 |
| 건설 XP | 건설 | 힘, 통솔 |
| 연구 XP | 연구 | 지능 |
| 사냥 XP | 사냥 | 힘, 민첩 |

### 레벨업 보상

| 레벨 | 보상 |
|------|------|
| Lv2~4, 6~9 | XP 1위 카테고리 관련 스탯 +1 |
| **Lv5** | 스탯 +1 + **후천 특성 1개** (폭주 이력 우선 → 행동 XP 기반) |
| **Lv10** | 스탯 +1 + **후천 특성 1개** (동일 로직, 2위 카테고리) |

### 히든 파라미터 (플레이어에게 비공개)

| 파라미터 | 의미 | 비고 |
|---------|------|------|
| **성장률** (스탯별) | 해당 스탯 상승 속도 | 같은 영웅도 매번 성장 속도 다름 |
| **상한선** (스탯별) | 해당 스탯 최대 도달치 | 재능의 한계 |

## 4. 사기 modifier 카탈로그 (RimWorld Thought 차용)

### 4-1. Memory modifier — 사건이 만들고 시간이 지우는

| Source (사건) | offset | TTL (턴) | 비고 |
|---|---|---|---|
| 전투 승리 | +5 | 3 | 원정/방어 모두 |
| 전투 패배 | -5 | 5 | |
| 동료 사망 | -25 | 20 | 다단계 감쇠 (10턴 후 -15로) |
| 부상 (knocked_out 회복) | -10 | 10 | |
| 유리한 판결 (이벤트 선택지) | +8 | 5 | |
| 불리한 판결 | -8 | 5 | |
| 연회 참석 | +6 | 3 | |
| 폭주 직후 (Catharsis) | +30 | 1 | 안전장치, 자동 부여 |
| 구원 시도 성공 | +20 | 5 | |
| 같은 영웅 폭주 목격 | -3 | 3 | 연쇄 반응 |
| 거점 시설 파괴 | -5 | 5 | 폭주/습격 결과 |

### 4-2. Situational modifier — 현재 조건이 켜고 끄는

| Source (조건) | offset | 활성 조건 |
|---|---|---|
| 침대 등급 | +0 ~ +5 | 숙소 시설 등급별 |
| 1인실 배정 | +3 | 병영 이상 + 영웅 수 < 정원 |
| 식사 등급 | -3 ~ +5 | 매 턴 자동, 식량 풍족도 기반 |
| 전장 배치 | -3 | 원정/방어전 진행 중 |
| 좋아하는 작업 | +5 | 시작특성 매칭 작업 수행 시 (예: 벌목꾼이 벌목) |
| 부상 상태 | -5 | injured 동안 |

### 4-3. Permanent modifier — 영구 (시작특성에서 부여)

| 시작특성 | offset | 비고 |
|---|---|---|
| 낙천적 (유나) | +12 | RimWorld Sanguine 직역 |
| 둔감한 (오린) | 0 (변동폭 -30%) | 사기 변동량을 0.7배로 |
| 예민한 (라엘) | 0 (변동폭 +30%) | 사기 변동량을 1.3배로 |
| 철의 의지 (가론) | 0 (폭주 임계 -5) | 임계값 자체를 변경 |

### Stack 규칙

- **stackLimit**: 같은 source 최대 누적 (기본 1, 일부는 2~3)
- **stackedEffectMultiplier**: 누적 시 효과 감소 (기본 0.75)
- 다단계 감쇠: 일부 강한 memory는 시간 지남에 따라 offset 약화 (예: 동료 사망 -25 → 10턴 후 -15 → 20턴 후 소멸)

> 이 카탈로그의 수치는 모두 `balance.csv` `morale_mod_*` 키로 이전 예정.

## 6. 영웅 관계 (bonds) — 요약

> **상세 SSOT**: [sin_system.md](sin_system.md) §2-A (계산 순서) + §3 (관계 매트릭스).
> 본 절은 영웅 데이터 구조 관점의 요약만 둠.

### 데이터 구조
```js
hero.bonds = { [otherHeroId]: bonds_min..bonds_max }  // asymmetric, 초기값 [balance.csv:bonds_initial]
```

- `bonds`: A→B와 B→A가 **다를 수 있음** (짝사랑/일방적 적대)
- 매트릭스 적용은 **두 영웅의 메인 죄종**으로 비교 (메인 고정이라 단순)
- 바알↔영웅 친밀도는 별도 축 없음 — 후천 특성(`acquiredTraits`)으로 이산 기록
- 핵심 원칙: 영웅은 봉신 아닌 용병 → 사기와 메인 죄종이 본질, 관계 변화는 사기 modifier로 반영

## 7. 낮 상태 · 밤 방어 구조 — 보존 (2026-04-20 확정)

### 낮 상태 3분류

```
[낮 상태]
├── 작업 중       → 생산적 행동, Stamina 소모
│    ├── 거점 내 (건설/연구/생산/채집) → 밤 방어 O
│    └── 거점 외 (원정/벌목/사냥)     → 밤 방어 X
├── idle          → Stamina 회복 (숙소 품질 배율), 밤 방어 O
└── 특수 행동     → 전용 효과, 밤 방어 X
     └── 병원 입원 (부상 전용, 치유 속도 가속)
```

### 밤 방어 자동 판정

```js
canDefend(hero) =
  hero.location === 'base'           // 거점 내
  && hero.stamina >= threshold        // 스태미너 임계 이상
  && !isInjured(hero)                 // 부상 아님
  && !isInSpecialAction(hero)         // 특수 행동 중 아님
```

- **명시적 "방어 배치" slot 제거** — Stamina가 자동 게이트
- **부상 영웅**: 뭘 하든 방어 불가, 병원으로 회복 가속 가능
- 임계 키: 방어 가능 Stamina = `[balance.csv:stamina_defense_threshold]` 이상
