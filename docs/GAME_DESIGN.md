## 1. 게임 개요

- **장르**: 거점 경영 시뮬레이션 (로스터 관리 + 자동전투 원정)
- **테마**: 7대 죄악 + 중세 다크 판타지
- **배경**: Aurea — 신이 걸었던 땅, 7대 죄악이 지배하는 세계 (TheSevenTactics 계승)
- **핵심 재미**: 7죄종이 만드는 **결함 있는 영웅들의 운영 트레이드오프** — 누구를 어디에 보내고, 누구의 만족을 채울지 매 턴 선택
- **컨셉**: "타락한 마왕이 결함 있는 인간들을 이끌고, 7대 죄악의 화신에게 복수한다"
- **스토리 전달**: 컷신이 아닌 **시뮬레이션 체감** — 챕터별 환경 변조 + 이벤트 + 보스 서사 ([chapter_scenario.md](game_design/chapter_scenario.md))
- **영웅 정체성**: 인간 (용병, 떠돌이, 버림받은 자) — 각자 성격적 결함(=죄종 수치)을 가진 결함 있는 인간들
- **플랫폼**: Phase 1 웹 프로토타입 (Phaser.js) → Phase 2 Godot(스팀)

## 2. 핵심 게임 루프

### 설계 의도: 통제 vs 변수

플레이어(바알)의 **통제**(낮)와 영웅의 **결함**(밤)이 충돌하는 구조.
- **낮**: 바알이 효율을 추구하며 명령 (통제)
- **밤**: 영웅들이 죄종에 따라 자율 행동, 사고 발생 (변수)
- **아침**: 밤의 결과를 보고받고 수습 + 새 이벤트 판결 (피드백)

### 하루(1턴) 구조

```
[아침] 밤의 결과 보고 + 소식 & 이벤트
  → ★ 밤 자율 행동 결과 보고 ("어젯밤 일어난 일" 텍스트 이벤트)
  → 문제 발생 시 플레이어 판결 (수습/처벌/무시)
  → 세계 소식, 영웅 간 사건, 죄종 관련 이벤트 발생

[낮] 거점 운영 & 원정 (바알의 통제)
  → 작업 지시: 건설 / 연구 / 생산 / 사냥 / 개척 등 → 투입할 영웅 선택
  → 채집/벌목: 식량/나무 획득 (맵 위 행동 팝업 — 프로그레스 + 결과 + 죄종 대사)
  → 사냥: 영웅 1명 → 야생 적과 1:1 전투 (맵 위 팝업)
  → 원정 파견: 영웅 선택 → 챕터 스테이지로 출발
  → ★ 방어 배치: 밤 습격에 참전할 영웅을 명시적으로 배치
  → 방어에 배치하지 않은 영웅은 밤 방어에 참여 불가
  ★ 핵심 딜레마: 영웅을 경제(건설/채집/사냥) vs 진행(원정) vs 생존(방어)에 어떻게 배분?

[저녁] 원정 결과 & 정보
  → 원정 결과 보고 (승리/패배, 전리품, 부상)
  → "더 진행시킬까, 귀환시킬까?" 판단
  → 감시탑 정보: 오늘 밤 습격 규모 예고 (감시탑 레벨에 따라 정보량 변동)

[밤] 자율 행동 + 적 습격 + 결산 (영웅의 변수)
  → ★ 영웅 자율 행동 — 죄종/체력에 따라 밤 이용 가능 시설을 자율 이용
    → 숙소 배치(사전 설정)에 따른 동실 영웅 간 상호작용
    → 죄종 드라마 발생 (텍스트 이벤트로 다음 아침 보고)
  → ★ 적 습격 — 3~5일 간격, 규모 가변 (소규모 정찰 ~ 대규모 공성)
  → 습격 있는 밤: 맵 위 오버레이로 방어전 (오토배틀 + 죄종 반응, 실시간 관전)
  → 습격 없는 밤: 자율 행동만 처리
  → 폭주/이탈 체크 → 연쇄 반응?
  → 다음 날로
```

### 플레이어 행동 요약

| 시간대 | 능동적 판단 |
|--------|-----------|
| 아침 | 밤 결과 수습 + 이벤트 선택지 고르기 |
| 낮 | ① 작업 지시 (건설/연구/생산/사냥/개척) + 투입 영웅 선택 ② 원정 파견 ③ **방어 배치** |
| 저녁 | 원정 계속/철수 판단 |
| 밤 | 습격 시 관전 + 죄종 반응 이벤트 (방어 배치 영웅만 참전) |

### 마을 안/밖 구분

| 위치 | 행동 | 밤 방어 | 챕터 진행 |
|------|------|--------|----------|
| **마을 안** | 건설, 연구, 생산, 채집, 휴식 | O | X |
| **마을 밖** | 원정, 벌목, 사냥 | X | O (원정만) |

### 메타 루프

```
[거점 성장] 건설 / 연구 / 생산 / 사냥 → 거점 강화 (낮: 통제)
    ↓
[원정] 영웅 파티 파견 → 챕터/스테이지 진행 → 전리품 (낮: 통제)
    ↓
[밤의 드라마] 영웅 자율 행동 → 죄종 기반 사건 발생 (밤: 변수)
    ↓
[방어] 3~5일 간격 습격 → 잔류 영웅으로 맵 위 방어전 + 증원 (밤: 변수)
    ↓
[수습] 아침 보고 → 판결/처벌 → 죄종 수치 변동, 폭주/이탈, 연쇄 반응 (아침: 피드백)
    ↓
반복 → 7챕터 클리어 → 엔딩
```

## 3. 바알(플레이어)의 역할

> 핵심 설계 원칙: **바알은 판결자이자 영주** — 국시(國是)로 거점을 통치하며, 이벤트 판결로 영웅들에 영향

### 구조

바알(플레이어)은 누적 수치 없이, **국시(Edict)** 를 통해 직접 통치 방향을 결정한다:
- 7죄종 중 1개를 **국시로 선포** (기간형 5턴 지속)
- 선포한 국시의 효과가 거점 전반에 적용 (전투력/자원/생산성/영웅 수치 변동)
- 만료 후 공백 기간(1턴)을 거쳐 다음 국시 선포 가능
- 매 턴 이벤트에 대해 선택지를 고르면 **영웅들의 죄종 수치**에 영향
- 트레이드오프 — 누군가를 만족시키면 다른 누군가가 불만

### 국시(Edict) 시스템 (2026-04-17 도입)

- **7종 국시**: 강병령(분노)/축재령(탐욕)/풍요령(폭식)/안식령(나태)/경쟁령(시기)/열정령(색욕)/위광령(교만)
- **기간형**: 선포 시 5턴 지속 후 자동 만료
- **자유 선포**: 해금 조건 없음. 언제든 7종 중 선택 가능
- **쿨다운**: 만료 후 1턴 / 조기 해제 시 2턴 / 대립 국시 전환 시 +1턴
- **효과**: 전투력/자원 획득/죄종 누적 가속/건설 속도 등 거점 전반에 배율/증감

> 상세: [sin_system.md](game_design/sin_system.md) §5 국시(Edict) 시스템

### 플레이어 선택이 게임에 미치는 영향

1. **영웅 죄종 수치 변동** — 선택에 따라 관련 영웅의 죄종 수치가 오르거나 내림
2. **연쇄 반응** — 한 영웅의 죄종 수치 변동이 다른 영웅으로 번져 새로운 사건을 만듦
3. **국시 선포** — 7종 중 1개 선택, 5턴간 거점 정책 적용
4. **거점 상태** — 시설 배치, 자원 배분 등 경영 판단의 직접적 결과
5. **원정 편성** — 누구를 보내고 누구를 남길지가 핵심 딜레마

## 4. 영웅 시스템

### 기본 구조

```
[영웅] — 반고정 생성 (이름+시작특성 고정, 스탯 랜덤, 죄종 수치 전부 0)
├── 이름 = "수식어" + 이름 (21명 고정 풀, 수식어: 현재 쌓인 죄종 상위 2개 조합)
│   └── 초기엔 쌓인 죄종이 없어 수식어 없음 (진행 중 부여/갱신)
├── 시작특성 1개 (영웅 이름별 고정, 장점 전용) — 정체성 + 죄종 쌓임 편향
├── 죄종 7수치 (분노/시기/탐욕/나태/폭식/색욕/교만, 각 **0~20 누적**) — 오염 게이지
│   └── **전부 0에서 시작**, 행동에 따라 쌓임. `topSin()`은 현재 상태일 뿐(고정 정체성 아님)
├── 후천특성 최대 2개 (Lv5, Lv10 레벨업 시 행동 XP or 폭주 이력 기반)
├── 레벨 (1~10) + 경험치 (카테고리별 누적: 전투/방어/노동/건설/연구/사냥)
├── 7스탯 (힘/민첩/지능/건강/감각/통솔/매력)
│   └── 총합 60~80, 최소 2, 최대 18 (죄종 무관, 랜덤 배분)
├── 외형: LPC 스프라이트 런타임 합성 (랜덤 파츠 조합)
└── 장비 3슬롯 — Phase 2
```

> **특성 3슬롯**: 시작특성 1 + 후천특성 2 = 최대 3개. 시작특성이 정체성 담당.

### 죄종 수치 시스템 (핵심) — 쌓임(누적) 프레임

영웅당 **죄종 7수치** (각 **0~20, 단방향 누적**). 모두 0에서 시작하여 행동마다 **쌓이는** 오염 게이지. 정화는 전용 수단(연회/야영/이벤트/특성 등)으로만. 어느 죄종이든 **18+ 도달 시 폭주**.

| 수치 구간(안) | 상태 | 효과 |
|-------------|------|------|
| 0~? | **깨끗** | 영향 없음 (좋은 상태) |
| ?~? | 발현 | 해당 죄종 소폭 효과 |
| ?~17 | **고양** | 파워 보너스 활성 + 문제 행동 시작 |
| 18~19 | **폭주 중** | 매 낮 페이즈 문제 행동 1회 |
| 20 + N턴 | **이탈** | 이탈 이벤트 → 영웅 손실 (N=3 현재 안) |

> 구간 경계값 확정 TBD. 낮은 수치 페널티 없음(깨끗 = 좋음).

### 폭주 (죄종 수치 18+)

어느 죄종이든 18+ 도달 시 `hero.isRampaging = true`. 해당 죄종에 따라 문제 행동 발동. 후천 폭주 특성이 있으면 특성 행동이 우선.

| 쌓인 죄종 | 폭주 문제 행동 |
|---------|-------------|
| 분노 | 랜덤 영웅 **도발** → 대상 영웅 wrath+1 |
| 교만 | 명령 **거부**(20%), 바알 결정 이의 이벤트 강제 |
| 탐욕 | 거점 자원 **횡령** (food-2 or gold-5) |
| 시기 | 강한 영웅 죄종 수치 **-1** |
| 폭식 | 매 낮 식량 **추가 소비** |
| 나태 | 행동 효율 감소(확률), 주변 sloth+1 **전파** |
| 색욕 | 집착 대상 영웅 **배치 거부**(20%) |

### 이탈 (죄종 수치 20 + 3턴 유지)

**가장 많이 쌓인 죄종**에 따라 이탈 방식 결정 (가중 확률).

| 쌓인 최고 죄종 | 이탈 방식 |
|-------------|---------|
| 분노 | 폭력적 탈영 |
| 교만 | "너 밑에 있을 수준이 아니다" 선언 후 이탈 |
| 탐욕 | 창고 털고 도주 |
| 시기 | 뒤에서 사보타주 후 이탈 |
| 폭식 | 보급품 갖고 이탈 |
| 나태 | 조용히 사라짐 |
| 색욕 | 다른 영웅 꼬여서 같이 떠남 |

### 폭주 → 타락 or 구원 (DD 고난/미덕 방식)

| 결과 | 확률 | 효과 |
|------|------|------|
| **타락** | 70% | 죄종 영구 강화. 더 강하지만 더 위험 |
| **구원** | 30% | 죄종 극복. 고유 능력 해금, 안정적 |

### 연쇄 반응 (죄종 쌓임의 파급)

한 영웅의 폭주 행동이 다른 영웅의 죄종 수치를 추가로 쌓이게 만드는 구조.

```
예시) 탐욕 영웅 폭주 → 자원 횡령
  → 자원 부족 발생
    → 폭식 영웅 gluttony +1 (허기로 인한 쌓임)
      → 나태 영웅 sloth +1 전파 (나태 전염)
        → 분노 영웅 wrath +1 (분노 유발)
```

### 죄종별 쌓임 조건 (쌓임 프레임)

각 행동이 어떤 죄종을 쌓이게 하는지. 감소는 **정화 수단**으로만 (카탈로그 TBD).

| 죄종 | 쌓이는 조건 | 정화 수단 |
|---------|-----------|----------|
| 교만 | 방어 승리, 보스 격파 | TBD |
| 탐욕 | 채집/원정 보상, 자원 부족 | TBD |
| 분노 | 전투/사냥/방어 배치 | TBD |
| 시기 | bench 중 타 영웅이 원정/보상 | TBD |
| 색욕 | 연회, 파티 동행 | TBD |
| 나태 | idle 1턴 유지 | TBD |
| 폭식 | 연회, 식량 풍부+idle | TBD |

> 정화 수단 카탈로그(연회/야영/시설/이벤트/특성 등)는 다음 세션 확정.

### 플레이어 행동 → 죄종 수치 변동

플레이어의 경영 판단이 영웅 죄종 수치를 오르내리게 한다.

| 플레이어 행동 | 죄종 수치 변동 |
|---|---|
| 원정/사냥/방어 배치 | 분노 수치↑ (호전적) |
| idle 유지 | 나태 수치↑, 분노 수치↓ |
| 연회 개최 | 폭식/색욕↑, 탐욕↓ |
| 채집 | 탐욕↑ |
| 방어 승리 | 교만↑ |
| bench 중 타 영웅 원정 | 시기↑ |

> 상세 설계: [hero_design.md](game_design/hero_design.md)

## 5. 거점(림보) 시스템

### 거점 = 게임의 무대

바알이 운영하는 거점 "림보". 건설/연구/생산으로 거점을 성장시키고, 원정으로 챕터를 진행한다.

### 자원: 3종

| 자원 | 획득 방법 | 주요 사용처 |
|------|----------|------------|
| **식량** | 채집 (거점 주변, 밤 방어 O) | 영웅 유지비 (턴당 소비), 연회 |
| **나무** | 벌목 (거점 외곽, 밤 방어 X) | 건설 (전 티어) |
| **골드** | 원정 전리품, 방어전 보상, 시장, 교역, 사냥 | 연구, 장비/소모품 구매, 영웅 고용 |

### 건설 (HOMM3 방식, 턴당 1개)

3단계 건설 트리. 영웅 1명 투입, 스탯(힘+민첩)에 따라 속도 차이. 비용은 **나무**.

| Tier | 시설 |
|------|------|
| 1 | 주점(영웅 고용), 대장간, 감시탑, 농장(식량 자동), 벌목장(나무 자동) |
| 2 | 병원(치료), 역마차(교역), 훈련장(영웅 HP 회복 속도 +50%), 감시탑 Lv.2 |
| 3 | 연금술소, 시장(수입), 감시탑 Lv.3 |

### 연구 (턴당 1개 진행, 다턴 소요)

영웅 1명 투입. 2계열: 경제(수입/건설), 영웅(죄종 이해/성장).

### 플레이어 행동 (낮)

| 행동 | 설명 | 제한 | 밤 방어 |
|------|------|------|--------|
| **건설** | 시설 신축/업그레이드 (나무 소모) | 턴당 1개, 영웅 1명 | O |
| **연구** | 기술 연구 (골드 소모) | 턴당 1개 진행, 영웅 1명 | O |
| **생산** | 장비/소모품 제작 | 시설당 1개, 영웅 1명 | O |
| **채집** | 식량 획득 (거점 주변) | 영웅 1명 | O |
| **벌목** | 나무 획득 (거점 외곽) | 영웅 1명 | X |
| **사냥** | 야생 적과 1:1 전투 (맵 위 팝업) | 영웅 1명 | X |
| **원정 파견** | 챕터 스테이지로 출발 | 영웅 1~3명 | X |
| **교역** | 상인 방문 시 구매/판매 | 상인 방문 시만 | - |
| **영웅 고용** | 영웅 1명을 보내 후보 데려오기 | 턴당 1명, 영웅 1명 투입 + 골드 (매력→할인), 최대 7명 | O |
| **국시** | 7죄종 중 1개 국시 선포 (기간형 5턴) | 쿨다운 없으면 언제든, 거점 전반에 효과 적용 ([sin_system.md](game_design/sin_system.md) §5) | - |
| **연회** | 폭식/색욕 수치 상승, 폭주 위험 | 5턴에 1회, 주점 필요 | - |
| **치료** | 부상 치료 / HP 회복 | 병원 or 소모품 | - |

### 영웅 투입과 마을 안/밖

```
영웅 7명
  ├── 마을 안 (건설/연구/생산/채집/치료/휴식) → 밤 방어 O
  └── 마을 밖 (원정/벌목/사냥) → 밤 방어 X
```

> 상세 설계: [base_design.md](game_design/base_design.md)

## 6. 이벤트/선택지 시스템

### 이벤트 = 게임의 심장

매 턴 1~2개 이벤트 발생. 플레이어의 판결이 영웅 죄종 수치 + 거점 상태를 바꿈.

### 이벤트 유형

| 유형 | 트리거 | 예시 |
|------|--------|------|
| **영웅 간 갈등** | 죄종 관계가 "대립"인 영웅이 같은 시설 배치 | [분노] vs [나태] 충돌 |
| **영웅 요청** | 죄종 불만 누적 | "원정 보내달라" ([분노] 욕구) |
| **폭주 대응** | 영웅 죄종 수치 18+ | [탐욕]이 창고 자원 횡령 시도 |
| **외부 사건** | 턴 수/챕터 진행 기반 | 이웃 세력 발견, 상인 방문 |
| **밤 습격 후유증** | 밤 방어전 결과에 따라 | 부상자 발생, 시설 파손 |

### 선택지 구조

모든 선택지는 **강력하지만 대가가 있는** 설계:

```
[이벤트] [분노]가 [나태]를 때렸습니다. 훈련을 빠졌다는 이유로.

▶ [분노]를 처벌한다
  → [분노] wrath-2, [나태] sloth-1 (처벌받은 측 인정)

▶ [나태]에게 참으라 한다
  → [나태] sloth-1, [분노] wrath+1 (억눌린 분노 누적)

▶ 둘 다 처벌한다
  → 둘 다 해당 죄종 수치 -1, 단기 질서 회복

▶ 무시한다
  → 가담 영웅 죄종 수치 +1씩 ("무법지대" 인식)
```

> 상세 설계: [event_design.md](game_design/event_design.md)

## 7. 원정 시스템

### 구조

던전 탐색이 아닌 **파견 방식**. 파티를 보내고 결과를 받는다.

```
파티 편성 (3명) → 목표 선택 (챕터/스테이지) → 파견
  → [낮 동안 자동 진행]
  → 저녁에 결과 보고
    → 전리품, 부상, 경험치
```

## 8. 전투 시스템

### 기본 구조: X축 오토배틀 + 죄종 반응

- **영웅이 알아서 싸움** — 자동 이동, 자동 공격 (X축, east/west)
- **플레이어(바알)는 죄종 반응으로 개입** — 전투 중 영웅 죄종에 따른 이벤트 발생 시 선택
- **죄종 기반 일기토** — 전투 중 영웅 성격에 따라 1:1 일기토 자동 발동
- 7스탯 기반 (원정: 힘+민첩, 방어: 힘+통솔)
- **영웅 HP 직격**: 영웅이 피격 담당 (HP = 50 + 건강×5)
- **스프라이트**: LPC 런타임 합성 (영웅 생성 시 랜덤 외형)

### 죄종 반응 시스템 (카드 시스템 대체)

- 전투 중 영웅 죄종 조건 충족 시 **죄종 반응 이벤트** 자동 발동
- 플레이어가 반응에 대해 선택 (격려/제지/허가/금지)
- 일기토 신청도 플레이어 판단으로 전환 (자동 발동 → 허가/금지)
- 전투 전 **작전 지시** (공격적/방어적/균형) 선택으로 사전 전략

### 전투 유형

| | 원정 전투 | 방어 전투 (밤 습격) | 사냥 |
|---|---|---|---|
| 편성 | 영웅 1~3명 | 잔류 영웅 전원 | 영웅 1명 |
| 진행 | 실시간 오토배틀 (상세 기획 예정) | **맵 위 오버레이** (MapDefenseMode) | **맵 위 팝업** (MapHuntPopup) |
| 개입 | 죄종 반응 이벤트 | 죄종 반응 이벤트 | 불가 |
| 증원 | 불가 | **가능** (대기 영웅) | 불가 |
| 빈도 | 파견 시 | **3~5일 간격** | 낮 행동 |
| 핵심 | 파티 편성 + 작전 지시 | 증원 타이밍 + 죄종 반응 판단 | 영웅 개인 전력 |

핵심 딜레마: 원정에 보낸 **영웅**은 밤 방어에 못 씀. 전투로 HP가 깎인 영웅은 거점 행동/차기 원정이 제약됨.

### 전투 공식 (balance.csv 기반)

```
영웅 HP = 50 + 건강(vitality) × 5  (기본 10 → HP 100)
HP 자연 회복 = 매 턴 10 (병원 시설로 배율 가중)
HP 0 → knocked_out → 귀환 후 'injured' → 회복 턴 소요

ATK(원정) = 힘×0.7 + 민첩×0.4
ATK(방어) = 힘×0.8 + 통솔×0.4
데미지 = ATK × (0.8~1.2 랜덤) × 버프배율 × (1 - 피해경감)
```

### 사망 시스템 (TheSevenTactics 계승)

| 상태 | 조건 | 효과 |
|------|------|------|
| 기절 | HP 0 | 귀환 후 회복 필요 |
| 중상 | 기절 + 추가 피격 | 장기 치료 (수 턴 출전 불가) |
| 사망 | 중상 + 추가 피격 | **영구 상실** |

> 상세 설계: [battle_design.md](game_design/battle_design.md)

## 10. 개발 범위 (Phase 1)

### Phase 1 목표: 핵심 재미 검증 (웹 프로토타입, 2~3개월)

**포함**:
- 이벤트/선택지 → 영웅 죄종 수치 변동 반영
- 영웅 7명 (랜덤 생성, LPC 스프라이트 합성) + 죄종 7수치 게이지 + 폭주
- 죄종 7수치(0~20 누적) + 시작특성(21명 풀, 장점 전용, 이름-특성 고정)
- 레벨/XP 성장 시스템 (1~10, 후천 특성 Lv5/Lv10)
- 체력(Stamina, 1~100) 게이지 — 건강 스탯 기반
- 연쇄 반응 시스템
- 거점 시설 23개 (Tier 0~3, 4카테고리)
- 개척 시스템 (3×3 → 5×5 확장)
- 숙소/격리 배치 + 밤 자율 행동 시스템
- 3자원 체계 (식량/나무/골드) + 채집/벌목/사냥
- 이벤트/선택지 30~40개
- 원정 1챕터 (분노, 스테이지 3 + 보스)
- 전투: 오토배틀+카드(방어전), 1:1 팝업(사냥), 결과형(원정 → 실시간 전환 예정)
- 1개 엔딩 (분노 루트)

**제외 (Phase 2 이후)**:
- 나머지 6챕터

## 죄종 간 관계 매트릭스 (TheSevenTactics 계승)

```
         교만  시기  분노  나태  탐욕  폭식  색욕
교만      ──   생성  강화   대립  강화   ·    강화
시기     전제   ──   생성   ·    강화   ·     ·
분노     강화  강화   ──   대립   강화   ·     ·
나태     대립   ·   대립   ──    대립  강화   대립
탐욕     강화  강화  강화   대립   ──   대립   강화
폭식      ·    ·    ·    강화   대립   ──   생성
색욕     강화   ·    ·    대립   강화  생성    ──
```

- **생성**: 앞의 죄가 뒤의 죄를 직접 낳음 (가장 강한 시너지)
- **강화**: 합치면 더 위험 (시너지)
- **대립**: 표면적 모순 (갈등, 단 극복 시 강한 보너스)
- **중립(·)**: 직접적 관계 없음
