1. 스토리텔러(Storyteller) 시스템

1.1 설계 철학

RimWorld의 스토리텔러는 게임의 "감독(Director)" 역할을 하는 AI다. 개발자 Tynan Sylvester는 이를 "AI Storyteller"라 명명했으며, 핵심 원칙은 다음과 같다:

  • 게임은 "이기는 것"이 아니라 "이야기를 만드는 것"이 목적
  • 플레이어가 너무 안정되면 위기를 주고, 너무 힘들면 숨을 돌릴 틈을 준다
  • 완전한 랜덤이 아니라 구조화된 드라마 곡선을 만든다

1.2 세 스토리텔러의 비교

스토리텔러철학이벤트 간격위협 스케일링특징
Cassandra Classic클래식 드라마 곡선예측 가능한 리듬 (4~8일)선형 상승 → 후반 급상승긴장→이완 교차, 대형 위협 후 강제 쿨다운

Randy Random순수한 카오스매우 불규칙 (0.5~15일+)분산(variance) 극대화카테고리 제약 약함, 연속 재앙/행운 가능

1.3 이벤트 선택 알고리즘

이벤트 선택은 다층 구조로 이루어진다:

Step 1: 틱(Tick) 기반 체크 — 매일 단위로 스토리텔러 호출
Step 2: MTB(Mean Time Between) 확률 계산
        P = 1 - e^(-checkInterval / mtbDays)
        예: mtbDays=10, 1일 체크 → 매일 약 9.5% 발생 확률
Step 3: 이벤트 풀 필터링
        전체 이벤트 → precondition 체크 → 쿨다운 필터 → 가중치 계산
Step 4: 가중치 기반 선택 (Weighted Random Selection)
        최종 가중치 = baseWeight × 상황별 수정자
Step 5: 스토리텔러별 조정
        Cassandra: 긴장 포인트 추적으로 균형 유지
        Randy: 카테고리 제약 완화

  • 이벤트 타이밍: 현재 "매 턴 1~2개 이벤트"에서, 긴장 누적기 방식을 도입. 연속 평화일에는 강제로 갈등/위기 이벤트 발생 확률 상승.
  • 거점 규모 스케일링: 거점 시설 수/영웅 수에 비례하여 밤 습격 규모 상승 → "성장이 곧 위험"의 긴장감.

  • 이벤트 가중치: 죄종 관계에 따른 가중치 — 분노 영웅 사기가 높으면 분노 관련 이벤트 가중치 상승.

2. Mood & Mental Break 시스템

2.1 Mood 계산 구조

RimWorld의 Mood는 기본값(50%) + 모든 활성 Thought의 Offset 합산으로 계산된다.

  • Mood 범위: 0% ~ 100%
  • 각 Thought는 완전히 독립적 — 서로 상호작용 없이 단순 합산
  • 현재 Mood는 목표 Mood를 향해 서서히 이동 (급격한 변동 방지)

Thought의 두 종류

종류설명예시
Memory사건 발생 시 생성, 시간이 지나면 소멸"동료가 죽었다" (-25, 20일)
Situational현재 조건 충족 동안 지속"아름다운 환경" (+5)

2.3 Mental Break 3단계

단계Mood 임계값Break 종류
Minor≤ 35%Sad Wander, Food Binge, Hide in Room, Insulting Spree
Major≤ 25%Tantrum(가구 파괴), Fire Starting, Corpse Obsession, Wandering
Extreme≤ 15%Berserk(아군 공격), Catatonic(무반응 1~7일), Murderous Rage

Break 발생 확률 곡선

Mood 35% (Minor 임계점): MTB ≈ 5~7일 (드물게)
Mood 30%:                MTB ≈ 2~3일
Mood 25% (Major 임계점): MTB ≈ 1~2일
Mood 20%:                MTB ≈ 0.5~1일
Mood 15% (Extreme):      MTB ≈ 0.3~0.5일
Mood 5%:                 MTB ≈ 1시간 이내 (거의 즉시)

핵심: 선형이 아니라 곡선 — 낮을수록 급격히 확률 상승

2.4 Inspiration (영감) — 긍정적 폭발

Mental Break의 정반대. Mood가 50% 이상일 때 확률적 발동 (MTB ≈ 7일).

Inspiration효과지속
Inspired Creativity예술/공예 품질 2단계 상승8일
Inspired Surgery수술 성공률 대폭 상승8일
Inspired Recruitment포로 회유 즉시 성공8일
Frenzy작업/전투 속도 대폭 증가8일
Shoot Frenzy사격 명중률 대폭 상승8일

2.5 Thought 스택 & 감쇠 규칙

  • stackLimit: 같은 Thought 최대 중첩 수. 기본 1 (새 발생 시 타이머 리셋)
  • stackedEffectMultiplier: 중첩 시 효과 감소 (보통 0.5~0.75). "무감각해지는" 효과
  • 다단계 감쇠: "동료 사망" → 0~10일: -25 → 10~20일: -15 → 소멸
  • 상반된 Thought 공존: "결혼"(+40) + "동료 사망"(-25) = 합산 +15
핵심 안전장치 — Catharsis: Mental Break 종료 후 Catharsis (+40 Mood, 1일)가 부여되어 Break 연쇄를 방지한다.
★ TheSevenSimulation 적용 제안
  • 사기 Modifier 시스템: 현재 사기(0~100)가 단일 수치인데, RimWorld처럼 복수의 독립 modifier 합산 방식을 도입하면 "왜 이 영웅의 사기가 낮은지" 플레이어가 파악 가능. 
    사기 = 기본값(50) + 갈망modifier + 판결modifier + 전투modifier + 관계modifier + ...
  • 확률 기반 폭주: 현재 사기 100=확정 폭주인데, 80 이상부터 확률적으로 폭주 체크하면 더 긴장감 있는 관리가 된다.
  • 카타르시스: 폭주 후 사기를 크게 내려서 바로 다시 폭주하는 무한 루프 방지.
  • 기억 시스템 (미설계 영역): "3일 전 처벌당한 기억"(-10, 5일 지속) 같은 Memory Thought를 도입하면 선택의 결과가 즉각+장기 효과를 모두 가짐.
  • Inspiration ↔ 구원: 현재 구원(30%) 시스템을 Inspiration처럼 고양 상태(71~99)에서 확률적으로 발동하도록 바꾸면 "폭주 전에 구원받는" 드라마가 가능.

3.1 Need 종류 총정리

Food0~1시간 경과 지속 감소음식 섭취
Rest0~1깨어있으면 감소수면
Joy0~1시간 경과 감소오락 활동

3.2 Joy 종류(Joy Kind) 시스템 — 핵심 설계

Joy Kind설명활동 예시
Solitary (고독)혼자 하는 활동명상, 독서, 산책
Social (사교)다른 사람과 함께대화, 파티, 체스
Dazzled (현혹)감각적 자극TV, 예술품 감상
Gluttonous (탐식)물질적 쾌락고급 음식, 음주
"같은 Joy만 반복하면 효과 감소" 설계:
  • Joy Kind별로 내성(Tolerance) 게이지를 따로 추적
  • 같은 종류 반복 → 내성 상승 → 효과 감소 (내성은 약 0.5/일 자연 감쇠)
  • "게임장 하나만 지으면 끝"을 방지 → 다양한 시설 건설 강제

3.3 Need → Mood 연결 구조

Need 수준        → 생성되는 Thought        → Mood 영향
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
매우 높음        → "[Need] satisfied"      → +5 ~ +10
보통            → (없음)                  → 0
낮음            → "[Need] unfulfilled"    → -3 ~ -8
매우 낮음       → "[Need] critical"       → -10 ~ -15

핵심: Need가 Mood를 직접 변경하는 것이 아니라, Need 수준에 따라 Thought가 생성되고, 그 Thought가 Mood에 영향. 간접적이지만 일관된 설계.

★ TheSevenSimulation 적용 제안
  • 갈망의 연속 게이지화: 현재 갈망이 "만족/보통/불만" 3단계인데, RimWorld Need처럼 0~100 연속 게이지로 바꾸면 더 세밀한 관리가 가능.
  • Joy Kind → 갈망 충족 다양성: "같은 방법으로만 사기를 올리면 효과 감소" 시스템 도입 가능. 
    분노 영웅에게 원정만 반복 파견 → 원정 만족도 내성 상승 → 사기 상승 효과 감소
→ 다른 방식(연회, 승리 보상 등)으로 충족해야 함
  • 시설 = 갈망 충족 수단: 주점은 Social Joy, 성당은 Solitary Joy, 훈련소는 분노 갈망 충족 등 시설-갈망 매핑.

4. Social & 관계 시스템

4.1 Opinion(호감도) 계산

호감도는 -100 ~ +100 범위의 수치로, 비대칭(A→B ≠ B→A)이다.

호감도 = 기본값(0) + 특성 보너스 + 대화 기억 + 사건 기억 + 외모 보정 + 관계 보정
카테고리요인수치 예시
특성 호불호Kind(친절) → 모든 타인에게+15
Abrasive(거칠음) → 모든 타인으로부터-25
대화즐거운 대화(Chat) 성공+5~+10
모욕(Insult)-15~-20
사건 기억"내 목숨을 구해줌"+15~+25
"내 연인을 죽임"-60~-80
외모Beautiful 특성+20~+40
Ugly 특성-15~-20
관계배우자(Spouse)+30 기본
라이벌(Rival)-25

4.2 관계 단계

              연인 → 약혼 → 배우자

4.3 Social Fight (사회적 싸움)

  • 발생 조건: 상호 opinion -80 이하, 모욕 후 일정 확률
  • 결과: 주먹 싸움 → 부상 → opinion 추가 하락 → 라이벌/적 관계 촉진
  • 드물게 사망 가능 (약한 폰이나 기존 부상 시)

    분노 A ↔ 나태 B: 기본 "대립"이지만
함께 원정 성공 3회 → 호감도 +30 → 대립 이벤트 확률 감소
반대로 갈등 반복 → 호감도 -50 → 대립 이벤트 격화
  • "같은 시설 배치" 상호작용: 같은 시설에 배치된 영웅 간 자동 상호작용 발생 (호감도 변동). 죄종 관계가 "대립"이면 갈등 이벤트, "생성/강화"면 친밀 이벤트.
  • Bond 시스템: 특정 조건(함께 보스 격파, 위기 구출 등) 시 영웅 간 "유대" 형성 → 한 쪽 이탈/사망 시 다른 쪽에 큰 사기 변동.

5. 특성(Trait) 시스템

5.1 기본 구조

  • 각 폰은 1~3개 특성 보유 (생성 시 결정, 변경 불가)
  • 특성은 mood, 작업 능력, Social, Mental Break 등 광범위하게 영향
  • 일부 특성은 스펙트럼(예: Industrious ↔ Lazy)으로 상호 배타

5.3 극단적 특성의 드라마 기여

Pyromaniac + Industrious + Kind: 평소에는 최고의 일꾼이자 모든 이에게 친절. 하지만 mood 하락 시 방화 → 본인이 만든 시설 전소 → 전체 mood 붕괴 → 연쇄 Mental Break
Chemical Fascination + Depressive: 우울 → 약물 의존 → 중독 → 금단증상 mood 하락 → 더 많은 약물 → 악순환
Tough + Brawler + Bloodlust: 근접 전투 무적, 킬마다 기분 상승. 하지만 평시에 Social Fight를 자주 일으켜 내부 분열

특수 위협

유형설명

7.2 레이드 규모 스케일링

Threat Points ≈ f(Colony Wealth, Population, Days) × difficulty × storyteller

적 구성: 총 포인트 내에서 유닛 "구매"
  - 부족민 (맨몸): 35pt / 부족민 (활): 55pt
  - 해적 (권총): 75pt / 해적 (라이플): 120pt
  - 기계체 랜서: 220pt / 기계체 센티피드: 400pt

성장 곡선:
  Day 1~15:   200~500pt   (1~3명)
  Day 15~60:  500~1,500pt (3~8명)
  Day 60~120: 1,500~4,000pt (8~20명)
  Day 120+:   4,000~15,000+pt (대규모)
Wealth Cap 전략: 숙련자는 불필요한 자산을 최소화하여 레이드 규모를 억제한다. "자산 관리"가 사실상 방어 전략의 일부.

7.3 전투 메커니즘

명중률 = 무기 정확도 × 사격 스킬 보정 × 거리 보정 × 조명 보정 - 커버 효과

커버 효율:
  벽/바위: 75% / 샌드백: 55~65% / 나무: 25% / 없음: 0%

갑옷 방어:
  실제 피해 = 원래 피해 × (1 - 방어도 × 랜덤(0.5~1.0))

8. 연쇄 반응(Chain Reaction) 메커니즘

8.1 시스템 간 연결 구조

[기분 시스템] ←→ [멘탈 브레이크] ←→ [사회 관계]
     ↑                ↓                    ↓
[건강/부상] ←→ [작업 시스템] ←→ [자원 생산]
     ↑                ↓                    ↓
[전투 시스템] ←→ [방어 시스템] ←→ [시설/건물]
     ↑                                     ↓
[이벤트/레이드]  ←→  [환경 시스템]  ←→  [온도/전력]

핵심: 독립적 시스템들이 공유 상태(shared state)를 통해 연결. 각 시스템은 단순하지만 연결이 복잡한 결과를 만든다.

8.2 대표적 연쇄 반응 사례

사례 1: 감정 폭발 도미노
레이드에서 식민자 A 사망 → A의 연인 B mood 급락(-30) → B Mental Break(방화) → 식량 저장소 소실 → 전체 식량 부족(-10) → C도 Break(폭주) → C가 D 공격 → D 부상 → 의료 자원 소모 → 추가 압박
사례 2: 전력 붕괴 연쇄
태양 플레어 → 전기 장치 정지 → 냉동고 정지 → 식량 부패 → 자동 포탑 비활성화 → 이 타이밍에 레이드 → 방어 실패 → 사망 스파이럴
사례 3: 전염병 + 레이드 콤보
역병 발생 → 3명 감염 → 의사가 치료에 전념(작업 감소) → 감염자 이동 불가 → 레이드 발생 → 건강한 인원만 방어 → 인력 부족 → 의사 부상 → 치료 중단 → 환자 사망 → mood 폭락 → 추가 Break
사례 4: 사회적 연쇄 붕괴
A-B 관계 악화 → Social Fight → B 부상 → A의 작업(요리) 중단 → 음식 품질 하락 → 전체 mood 하락 → C Break(은둔) → 생산성 급락 → 방어 공사 미완성 → 레이드 취약
사례 5: 겨울 기근 스파이럴
화산 겨울 → 작물 감소 → 비축 소모 → 사냥 시도(야생 동물도 감소) → 가축 도축(Bond 동물 -8) → mood 폭락 → Break → 악순환 → 인육 섭취(-20, Cannibal은 +20)

확장: 폭주 결과 → 자원/시설 피해 → Need 변동 → 다른 영웅 사기 변동
                                                → 방어력 약화 → 습격 피해 증가
                                                → 갈망 변동 → 추가 이벤트
  • 연쇄 횟수 제한: 1턴 내 연쇄 반응은 최대 3회로 제한 (무한 루프 방지).
  • 카타르시스 도입: 폭주 후 사기를 50으로 리셋 + 2턴간 "정화" 상태(폭주 면역).
  • 긍정적 연쇄 설계: 현재 부정적 연쇄만 있음. 다음과 같은 긍정 연쇄 추가: 
    원정 대승 → 전체 사기 +10 → 고양 상태 영웅의 구원 확률 상승
  → 구원 발생 → 전체 사기 추가 상승 → 시설 효율 보너스
  • "공유 허브" 설계: RimWorld에서 Mood가 모든 시스템의 허브인 것처럼, 사기(Morale)를 모든 시스템의 중심 연결 고리로 명확히 설계.

10. 난이도 & 리플레이

10.1 "Losing is Fun" 설계 철학

RimWorld의 핵심 사상:

  1. 실패는 콘텐츠다: 주민의 죽음, 콜로니의 몰락은 이야기의 일부
  2. 완벽한 방어는 없다: 킬존이 완벽해도 드롭포드/인페스테이션이 돌파
  3. 부유함 ≠ 안전함: Wealth 상승 → 위협 증가. "성장이 곧 위험"
  4. 연쇄 반응의 극적 효과: 데스 스파이럴이 가장 극적인 이야기를 만듦
  5. Commitment Mode: 저장/불러오기 금지로 결정의 영속성 보장

10.2 시나리오 시스템

시나리오시작 조건특징
Crashlanded3명, 중간 장비표준 시작
Rich Explorer1명, 고급 장비인력 부족 챌린지
Lost Tribe5명, 열악 장비기술 낮지만 인력 풍부
Naked Brutality1명, 장비 없음극한 챌린지

시나리오 에디터로 완전 커스텀 가능: 시작 인원/아이템/특성, 강제 이벤트, 건설 금지 등.

10.3 절차적 생성 요소

  • 세계: 구체 형태, 지형/기후/팩션 시드 기반 생성
  • 맵: 착륙 위치별 지형/자원/동굴/유적 무작위
  • 주민: 이름/외모/배경/스킬/특성 조합 무작위
  • 관계: 레이드 적이 주민의 가족일 수 있음 → 드라마
  • 이벤트: 적응형 AI가 콜로니 약점을 파악하여 이벤트 선택
★ TheSevenSimulation 적용 제안
  • "패배해도 재미있는" 설계: 영웅 전멸/거점 함락을 "게임 오버"가 아니라 "이 회차의 이야기 결말"로 프레이밍. 패배 시에도 발생한 드라마를 요약하는 "회고록" 화면.
  • 리플레이 변수: Phase 2에서 시작 영웅 조합/초기 자원/챕터 순서를 변경 가능하게 하면 리플레이 가치 상승.
  • 적응형 습격: 거점의 약점을 AI가 분석 — 성벽 강하면 지하 침투, 영웅 적으면 대규모 파상, 자원 풍부하면 약탈 집중.

11.4 RimWorld과 TheSevenSimulation의 핵심 차이점

RimWorld 시스템을 그대로 이식할 수 없는 구조적 차이를 인식하고 변형해야 한다:

차이점RimWorldTheSevenSimulation변형 방향
규모식민자 3~20명영웅 7명 (고정)적은 수로 더 깊은 개인 드라마. 1명의 손실이 치명적.
전투실시간 직접 조작완전 자동 (결과만)전투 자체보다 "누구를 보내는가"의 편성 판단에 집중.
시간연속 시간 흐름턴 기반 (하루=1턴)modifier 지속을 "턴" 단위로 관리. 감쇠도 턴 기반.
테마범용 SF 서바이벌7대 죄악 다크 판타지죄종이 특성+Need+Break를 모두 통합하는 강력한 태그.
플레이어 역할전지적 관리자바알(판결자)"선택지"를 통한 간접 관리. RimWorld보다 적은 마이크로.
최종 핵심 메시지:
RimWorld의 진짜 교훈은 개별 시스템의 복잡함이 아니라 "단순한 시스템들이 공유 상태를 통해 연결될 때 창발하는 드라마"이다.

TheSevenSimulation에서 사기(Morale)를 RimWorld의 Mood처럼 모든 시스템의 허브로 설계하면, 죄종·갈망·판결·전투·관계가 하나의 수치를 통해 서로 영향을 주고받으며 예측 불가능한 드라마를 자동으로 생성할 수 있다.
