# 기획 리뷰어

TheSevenSimulation의 **기획 전체를 냉정하게 평가**합니다.
4개 전문가 에이전트를 병렬로 실행해 도메인 내 문제를 깊이 분석하고, 이 에이전트(Opus)가 교차 충돌을 종합 판정합니다.

---

## Phase 1: 전문가 병렬 분석

**4개 에이전트를 동시에(단일 메시지) 스폰합니다. 순차 실행 금지.**

## Phase 2: 교차 충돌 종합 (Opus — 이 에이전트가 직접 수행)

4개 에이전트 결과를 모두 받은 뒤 다음 순서로 종합한다.

### 2-1. 가정 충돌 탐지
각 에이전트의 "다른 시스템에 대한 가정" 항목을 비교한다.
- A가 가정한 것을 B가 다른 내용으로 가정하면 → **충돌** 로 표시
- 가정이 실제 기획서와 다르면 → 해당 문서의 관련 섹션을 직접 읽어 확인

### 2-2. 질문 응답
각 에이전트가 던진 질문에 대해:
- 다른 전문가 보고서에서 답이 있으면 → 인용
- 없으면 → "기획서에 미정의" 로 표시

### 2-3. 최종 판정 출력

```
## design-review 결과

### 잘 된 점 (2~3줄 이내로 요약)

### 도메인별 주요 문제
#### 전투/원정
#### 경제/거점
#### 영웅/시스템
#### 내러티브/UX

### 교차 시스템 충돌 (핵심)
- [충돌 A] 전투가 가정: ... / 경제가 실제: ...
- [미정의] 어느 문서에도 명시되지 않은 것

### 재미 관점 총평
"이 기획은 실제로 재미있을까?" — 냉정하게 판단

### Phase 1 범위 적합성
2~3개월 웹 프로토타입으로 구현 가능한가?

### 우선 해결 제안 (Top 3)
1.
2.
3.
```

## 실행 규칙
- Phase 1의 4개 에이전트는 반드시 **단일 메시지에서 동시에** 스폰 (병렬)
- Phase 2는 4개 결과를 모두 받은 뒤에만 시작
- **문서 수정은 사용자가 명시적으로 요청할 때만** 진행
- 칭찬보다 **문제점과 충돌**에 집중
