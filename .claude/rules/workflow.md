# 구현 워크플로우

## 0단계: 기능 확인
구현 시작 전, 해당 기능의 범위를 사용자와 확인한다.

## 1단계: 개발 계획 생성
- **목적** — 기능/목적 한 줄 요약
- **작업 유형** — 신규 개발 / 리팩토링 / 버그 수정
- **변경 파일** — 신규/수정/삭제 파일 목록
- **구현 순서** — 단계별 작업 순서
계획 제시 후 **사용자 승인**을 받는다.

## 2단계: 개발 계획 검증
4대 관점: 아키텍처 정합성, 데이터 안전성, 에러 처리, 유지보수성

## 3단계: 파급 영향 검토
계획대로 진행했을 때 **연쇄적으로 수정이 필요한 부분**을 사전에 파악한다.
- **기획서 정합성** — 변경 내용이 다른 기획 문서(GAME_DESIGN.md, 세부 설계)와 충돌하지 않는가? 함께 업데이트할 문서는?
- **코드 의존성** — 변경하는 모듈을 참조하는 다른 코드가 있는가? (import, 함수 호출, 데이터 구조)
- **데이터 연동** — CSV/Store 키 변경 시 로드/저장하는 곳이 모두 대응되는가?
- **UI 반영** — 시스템 변경이 화면(ui_design.md, 씬 코드)에 반영되어야 하는가?

파급 영향 목록을 작성하고, 구현 시 함께 처리한다.

## 4단계: 구현
검증과 파급 검토를 통과한 계획대로 코드를 작성한다.

## 5단계: 테스트
- 기능 테스트 (성공 + 에러 케이스)

## 6단계: 결과 저장 및 문서 업데이트
- DEV_PLAN.md 상태 갱신
- 3단계에서 파악한 파급 문서 업데이트 확인
- 변경 사항 보고
- **`docs/session-state.md` 업데이트** — 완료된 작업, 주요 결정, 다음 할 일 기록
